Configuring the GAWizard
2D Reference Help > Tools > GAWizard > Configuring the GAWizard

To add the GAWizard to your simulation model:

  1. Click the toolbar Tools and insert the GAWizard into your simulation model. Double-click the icon to open the dialog of the GAWizard.

    You can define the optimization problem with the GAWizard by using:

    • Drag-and-Drop: Hold down Shift and drag the tables, the sequence of which you would like to optimize onto the GAWizard and drop it there. When you would like to optimize an attribute of an object, hold down Shift and drag this object onto the GAWizard, drop it and select the attribute. Then, you can change the value range of the parameters to be optimized into the table ProblemDefinition. To open this table, click the button Open next to Definition of the optimization parameter on the tab Define. Drag the object, whose attributes and methods you would like to use for evaluating the created solutions, onto the GAWizard and drop them there. Then, select the attributes and methods of the object. To weight these values differently, click Open next to Fitness calculation by table.

    • When you use GA-tables for describing your optimization problem, hold down Shift and drag these tables onto the Wizard and click Apply. This registers all GA-tables with the GAWizard. Define a Method as the configuration method, The attribute individual designates the table of the individual that is to be evaluated and that the genetic algorithm created. The first line of the source code shows how you can access the chromosomes of the individual. With the configuration method you can transmit all properties of the chromosomes to your simulation model. The result of the method is true when a parametrization can be executed with the values of the individual. When the parametrization fails, the respective individual will be assigned a bad fitness value, a so-called penalty value. The penalty value for unrealizable individuals is the attribute penaltyValue of the GAWizard. To change the value of this attribute, select the GAWizard in the Frame, into which you inserted it, and press the F8 key.

  2. Select the direction of the optimization: Minimum or Maximum. Enter the Size of generation and the Number of generations on the tab Define.

  3. You can determine the fitness value with the GAWizard by using:

    • The table Fitness. Select the radio button Fitness calculation by table on the tab Define and click Open. This opens the table Fitness. Enter the names of the values into the cells of the column Target value. When the fitness value is determined by several target values, enter their weighting into the cells of the column Weighting.

    • You can let a method calculate the fitness. To do so, select the respective radio button, insert a Method, and click Apply. If you want to, the Wizard can prepare a Method for calculating the fitness. The return value is the fitness value and has to have a numeric data type, such as real or time for example

  4. Click Reset and Start on the tab Run to start the optimization.

You can stop the optimization run after the individuals of the active generation have been completely evaluated and then modify the settings and the parameters of the object GAWizard. During the simulation run, you can click Stop on the tab Run. As long as the program is still evaluating the individuals of the active generation the button shows Wait. When the button shows Start again you can click it to continue the optimization run for creating new generations.

After the program is finished with the optimization run, it shows a dialog with the time the optimization run took. The wizard transfers the input values for the best solution it found into the simulation model. You can now analyze how the optimized model behaves.

To start additional optimization runs, click Reset and Start on the tab Run or click the icon of the GAWizard with the right mouse button and select Reset or Start and Stop on the context menu.

The number of required simulation runs for an optimization can become quite large value.

For a stochastic simulation the individuals should be evaluated by several simulation runs. You can enter the number of simulation runs for evaluating an individual on the tab Define > Observations per individual . The number of individuals to be evaluated results from the Number of generations and from the Size of generation. You can enter both values on the tab Define as well.

In the first generation Plant Simulation evaluates the number of generations which you entered as the Size of generation. In each of the following generations it has to evaluate twice as many individuals. When, for example, you use 3 observations for evaluating an individual and create 10 generations with a frequently used Size of generation of 30, Plant Simulation has to execute 1710 simulation runs (3 * (30 + 2 * 30 * 9)).

The number of simulation runs, which the GAWizard executes, results from this formula:

Number of simulation runs = observations per individual * (generation size + 2 * generation size * (number of generations - 1))
