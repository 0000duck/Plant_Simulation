writeObjectFile
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Importing and Exporting Data in Text Format > writeObjectFile

Syntax:

 
Path.writeObjectFile(FileName:string [,NoDebugger:boolean]) → boolean

The method writeObjectFile exports data of a list or table to the designated file of type .obj. Plant Simulation overwrites any data that this file might already contain.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

Parameters

  • The parameter FileName of data type string designates the name of the object file into which Plant Simulation exports the data.

  • The optional parameter NoDebugger of data type boolean sets if the Debugger opens after an unsuccessful access or not.

    When you do not specify the optional parameter of data type boolean, the Debugger always opens.

    When you enter true, the Debugger does not open and the method returns the value false when an error occurs.

Return Value

The return value has the data type boolean.

Example:

 
MyStackFile.writeObjectFile("C:\users\johnE\times.obj")
MyTableFile.writeObjectFile("C:\temp\run1.obj")
if MyCardFile.writeObjectFile("C:\temp\data.obj", true) 
   -- data written successfully
else -- take appropriate measures
end
