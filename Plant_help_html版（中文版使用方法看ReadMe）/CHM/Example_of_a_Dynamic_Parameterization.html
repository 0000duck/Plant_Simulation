Example of a Dynamic Parameterization
2D Reference Help > Tools > ExperimentManager > Advanced Settings > Example of a Dynamic Parameterization

Below we’ll demonstrate how to use rule-based parameterization by means of the inventory model by Law and Kelton, compare Use a Two-level Experimental Design and Use Factorial Analysis. The experiment design showed that the least cost-intensive order strategy for the store consists of the order type Normal and the inventory-evaluation interval of 1 month . We want to investigate the inventory costs for all values ranging from 0 to 70 with a step size of 5 for the values safety stock and the difference. Note that we do not use the experiment table in this example.

Now we have to create rules which set these pairs of values ( safety stock and difference ):

(0,0), (0,5), (0,10), … , (0,70), (5,0), (5,5), (5,10), … , (5,70), (10,0), (10,5), (10,10), … , (70,70)

This results in 152, i.e., 225 experiments. In the first experiment an initialization rule sets both values to 0. In a second rule we check if the value difference is less than or equal to 70. If this condition is met, the value difference is increased by 5. A third rule checks if the value 70 for the difference is exceeded. When this is the case the corresponding action resets the value difference to 0 and increases the value safety stock by 5.

The second and the third rule only work when the second rule is evaluated first. For this reason you have to assign a greater priority to the second rule than to the third rule.

Then, we set the Number of experiments to 225 on the tab Rules. Finally we start the experiment run by clicking Start.

Showing the results in a user-defined chart

To open this model and to run experiments with it, click the Window ribbon tab, click Start Page > Getting Started > Example Models and click Small Examples. Then, select the respective Category, the Topic, and the Example in the dialog Examples Collection and click Open Model.

If you want to show the results in a chart as a 3D Surface, you have to use the evaluation method in the sample model.
