Working in the Method Window
2D Reference Help > Information Flow Objects > Method > Working in the Method Window

Double-click the icon of the Method, which you inserted into a Frame, to open its dialog window. To change the properties of the Class of the object, double-click it in the Class Library or on the tab Information Flow in the Toolbox. It provides access to the method editor and to the Method Debugger.

  • Click Debug on the Extras ribbon tab or press F11 to change from the method editor to the method debugger.

  • Click any of the Step buttons on the debugger toolbar repeatedly or once to return to the editor.

  • To select a single word, double-click it. To select an entire line, triple-click anywhere in the line.

  • To select all text in the window, press Ctrl+A or click the mouse button four times.

  • To copy the selected text, press Ctrl+C, to paste text from the clipboard, press Ctrl+V.

  • To move the selected text, drag the mouse to a different location, and drop it.

  • To undo the last changes you made to the source code, click or press Ctrl+Z .

  • To restore (redo) the source code to the text it had before an undo action, click or press Ctrl+Y.

  • To insert a line break, enter a backslash \ if the source code is too long to fit on one line. This makes it easier to read the source code in the method window while Plant Simulation still interprets the source code as a single entity. 

    priority:= "Urgent"
very_long_text := "Frank Jones\
                   Halford Lane 9\
                   Cedar Rapids, IA 52409"
Text := "It is \"very\" urgent." -- outputs It is "very" urgent.

  • To scroll horizontally in the Method Editor, hold down the Shift key and roll the mouse wheel.

  • To enter a comment explaining the intention of your source code, enter two hyphens -- at the beginning of the line and then type in the comment. The Method displays comments in green. The only exception are strings in which you cannot enter a comment within the quotation marks. Instead of two hyphens (--) you can also start a comment with two forward slashes (//).

    Comments improve the legibility of a Method and help to understand the intentions behind your programming. A comment is especially helpful in a long Method, when you or a colleague look at it again at a later point in time to extend its functionality or to debug it. We encourage you to make extensive use of comments. Comments do not affect the simulation speed at all!

    You can insert a comment anywhere, except within a string value of your program. This means that you cannot enter a comment within the quotation marks that designate a string.

    • To start a one-lined comment, enter two hyphens --, and type in your remarks. This comment ends at the end of the line.

    • To start a comment that spans several lines, enter a forward slash and an asterisk /*, and type in your remarks. To end the comment, enter an asterisk and a forward slash */.

  • To execute the source code of the Method, click Run Method or press F5 or click Debug Method or press F11.

  • To increase or decrease the font size in the window of the Method Editor or the Method Debugger, hold down the Ctrl key and roll the mouse wheel forward and backward.

    To decrease the font size, hold down the Ctrl key and roll the mouse wheel backward. When you change the font size with the mouse wheel, Plant Simulation uses the changed font size until you change it the next time or until the end of the Plant Simulation session for all Methods which you open afterwards. Plant Simulation uses the changed font size in all open Method windows, not just in the one in which you rolled the mouse wheel.

    Instead of rolling the mouse wheel, you can also use these key combinations:

    Ctrl plus + increases the font size

    Ctrl plus 0 resets the font size to the standard size of 10 points

    Ctrl plus - decreases the font size

  • Select if you want to show line numbers, if you want to highlight keywords and what kind of indentation you want to use on the Edit Ribbon Tab and on the Tools Ribbon Tab in the present instance of the Method.

  • Select how Plant Simulation shows SimTalk code in all Method objects under File > Preferences > Editor in the Plant Simulation window.

An asterisk ( *) in the title bar of the Method window shows that the Method contains changes that have not been applied yet. To do so, click Apply changes on the Edit Ribbon Tab.

The Context Menu of the Method also provides the commands Toggle Class Breakpoint, Toggle Instance Breakpoint, Run to Cursor, Set Next Statement, and Open Object.
