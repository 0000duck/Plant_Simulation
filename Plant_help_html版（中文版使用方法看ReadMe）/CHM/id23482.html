Segments
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Dialog Items of the Objects > Tab Curve > Segments

To open the Segments table,which shows information about the starting point and the curved and straight segments of the curved object, which you inserted, click this.

You can edit the tangential angle, the length, the curve angle, the radius, the vertical curve and the z-difference of a segment, while Plant Simulation computes the remaining, grayed out values.

If the curved object is an animation polycurve in 3D, you can define a tangential angle and a length in addition to also place this starting point outside of the zero point of the object in the x-direction and the y-direction.

You can view how the table has to be formatted, which you can assign with the method setCurveSegments.

The definition of the segments table always shows the original orientation of the segments without a possible rotation or mirroring. The rotation is stored in the attribute ObjectAngle and mirroring is stored in the attribute ObjectMirrored. This allows to keep inheritance of the definition of the segments even when one or more segments are rotated or mirrored.

The first row shows the additional offset in the z-direction to the Base height in order to define the starting point of the curved object. Enter a different value to increase the distance of the feet of a conveyor from the floor of the plant.

The column

Shows and/or Sets the

Tangential Angle

change of the tangential angle of a segment, relative to the previous direction of the segment. For the starting point it is relative to the horizontal. A positive number designates a clockwise angle, a negative number a counterclockwise angle. The tangential angle is located on the plane.

Length

length of a straight segment, i.e., of the distance between the start and the end of the segment.

When the straight segment contains an incline, i.e., when the z-heights of the starting point and of the end point differ, the length is the length that is projected onto the xy-plane.

A curved segment does not show any value here.

Curve Angle

angle α that is formed by the radius between the start and the end point of a horizontal or a vertical arc segment. A positive number designates a clockwise angle, a negative number a counterclockwise angle.

Radius

radius of a horizontal or a vertical arc segment. A straight segment does not show any value here.

Vertical Curve (only recognizable in 3D)

state of the segment. If the segment is a vertical curve segment, the check box is selected. If it is no vertical curve segment, it is cleared. For a straight segment and a horizontal curve segment, the check box is cleared.

When you insert a vertical curve segment in 3D, the check box is selected. You can clear the check box to change the vertical curve segment to a horizontal curve segment.

Note that a vertical curve segment affects its neighboring segments more than it does other segments. For all other segments the tangential angle only affects the succeeding segment. A vertical curve segment, in addition, enforces that its own tangential angle and the tangential angle of the succeeding segment is 0 and that the preceding as well as the succeeding segment also tangentially connect in the vertical direction. This means that a succeeding vertical curve segment will be interpreted depending on its preceding segment and that Plant Simulation enforces a suitable relation of projected height and height offset for all other succeeding segments.

ΔZ

additional offset to the base height to the predecessor segment.

X

x-coordinate of the anchor point of a segment in the Frame. Plant Simulation computes this value.

Y

y-coordinate of the anchor point of a segment in the Frame. Plant Simulation computes this value.

Z

z-coordinate of the anchor point of a segment in the Frame. Plant Simulation computes this value.

ΔL

length of the object up to this segment

A button that looks like this means that inheritance is active. Then, the object inherits, i.e., uses, the shape of the origin object from which you derived it. When you modify the value of the origin object, the program also modifies the value of the derived object.

A button that looks like this means that inheritance is not active. The values only apply to the selected object.

The segments table shows absolute coordinates, while Plant Simulation stores the anchor points as relative coordinates, which relate to the first anchor point. The position of the curved object in the Frame is determined by the attributes xPos and yPos, which are the respective coordinates of the first anchor point. If the segments table is inherited this means that the relative coordinates of the anchor points are inherited.

For this reason it may happen that an inherited segments table shows other coordinates than the original curved object when the xPos and/or yPos of the curved object has changed. Inheritance relates to the shape of the curved object, not to the position at which it is inserted in the Frame!
