InfoDriver
Add-Ins Reference Help > Database Interfaces > ODBC > Read-Only Attributes of the ODBC Interface > InfoDriver

Syntax:

 
Path.InfoDriver

The read-only attribute InfoDriver returns the table Driver.

Entry

Description

Driver Name

Name of the driver

Driver Version

Version of the driver

Driver ODBC Version

ODBC version which the driver supports

ODBC Version

ODBC version which the Driver Manager supports

API Conformance

The API Conformance Levels define a fixed functionality which the driver can realize to meet a defined level. Possible values are None, Level1, or Level2

SAG Conformance

Shows if all functions, which are set by SAG (SQL Access Group), are supported by the driver. Possible values are SAG compliant or Not SAG compliant.

SQL Conformance

Shows the SQL grammar which the driver supports. Possible values are Minimum, Core, or Extended.

File Usage

This row is of importance for Single-Tier-drivers only. It shows if the driver treats a file as a table or as a data base. Possible values are Not supported, Table , or Qualifier.

Number Connections

Shows the maximum number of concurrently active connections. The value 0 stands for an unlimited number of connections.

Number Statements

Shows how many instructions can be active at the same time for this driver. The value 0 stands for an unlimited number of instructions.

Return Value

The return value has the data type table.

Example:

 
TableVariable := ODBC.InfoDriver
