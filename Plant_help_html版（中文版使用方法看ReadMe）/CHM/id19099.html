execute
SimTalk Reference > General SimTalk Functions > Predefined Functions > Miscellaneous Global Functions > execute

Syntax:

 
execute(SourceCode:string [,Parameter:parameter, ...])

The function execute receives a string and interprets it as though it were the source code of a method.

The function execute accepts all statements. The anonymous identifier ? refers to the Method object that called the execute function. The anonymous identifier @ is received by the calling method.

The semantics of execute is as follows:

  • When a runtime error occurs while the string method is being executed, there are two possibilities:

  • When the setting Ignore Errors in Formulas is deactivated, Plant Simulation opens the Debugger.

  • When the setting Ignore Errors in Formulas is activated, the function execute acts like the function executeSilent.

  • When no runtime error occurs:

    • If the string method does not have a return value, execute returns VOID.

      Example:

      		 
      var objPath : string := "Track"
var attrName: string := "Length"
var attrVal : length := 3.5
execute(to_str(objPath, ".", attrName, ":=", attrVal))

    • In all other cases, execute will return the return value of the string method.

      Example:

      		 
      execute("-> integer; return 42") -- returns 42

    • You can also pass parameters to the execute statement.

      Example:

      		 
      execute("param x, y: integer; print x+y", 1, 2) -- outputs 3

The function execute executes the referenced Method. Use the function execute to execute methods that you referenced via variables. Let us say you save a method to a variable obj of data type object, you can then call this method using obj.execute.

The syntax which is used in the argument for the execute function has to be the same as the Method syntax which calls the execute function.
