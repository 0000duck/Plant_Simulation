Import File
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Window of Lists and Tables > List Ribbon Tab > Import File

The command Import File imports a list, which you saved beforehand. It opens the dialog window Open. Double-click the name of a list/table which you exported before with the commands Export Text File or Export Object File or click the name once and then click Open.

You can open files of these types:

Plant Simulation automatically recognizes the encoding of a text file when opening it.

  • Opening an Plant Simulation object file

    An Plant Simulation object file contains the contents and the Plant Simulation-format of the list. This way you can exchange lists between different Plant Simulation simulation models. Plant Simulation opens it with all values and formats, which you saved with the extension .obj.

  • Opening a text file

    A text file only contains the contents of the list. You can also create a text file in a text editor or spreadsheet application and then open it as a *.txt file in an Plant Simulation list, compare Text File Format. Click a file you saved with either the extension .txt or with no extension.

  • Opening an Excel workbook

    Plant Simulation uses Excel as COM-server when reading and writing MS Excel-files. For this reason MS Excel has to be installed on your computer. To continue using the previous, no longer supported Excel interface, you can use the start option -NativeExcel.

    When Plant Simulation reads an MS Excel table, it attempts to convert the values in the individual column to the data types of the columns in your Plant Simulation table. This only works, when you created the columns on the Excel worksheets so that they only contain a single data type, meaning that, when you, for example, assign the data type String to a cell of a column, the entire column may only be of data type String.

    Row 0 (zero) is the exception: When the table, which Plant Simulation reads, has a column index, it interprets row 0 as the column index and it will not be part of the data type designation of the columns.
