transportOrder
2D Reference Help > Tools > CrossSlidingCar > Methods of the CrossSlidingCar > transportOrder

Syntax:

 
Path.transportOrder(SourceObject:object,TargetObject:object,Part:object,Priority:integer)

The method transportOrder enters a transport order into the internal orders table ( OrderList ) of the cross-sliding car. The table of transport orders looks like this:

Parameters

  • The parameter SourceObject of data type object contains the object at which the part is to be picked up, i.e., the source object.

  • The parameter TargetObject of data type object contains the object to which the part is to be delivered, i.e., the target object.

  • The parameter Part of data type object contains the part.

  • The parameter Priority of data type integer contains the priority. It is not used at the moment because the parts are processed in the order in which they were entered into the orders table.

  • Example:

    		 
    var part : object := @
var PartLocation : object := ?
var PartTarget : object := part.destination
var Priority: integer := 1
CSC1.transportOrder( PartLocation, PartTarget, part, Priority )
