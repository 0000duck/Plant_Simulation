delete
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Accessing Lists and Tables > delete

Syntax:

 
Path.delete
Path.delete([Range:listrange, ...])

The method delete deletes the contents of the list object designated by Path.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

You can also delete sub-tables, which inherit their contents, with the method delete. The inheritance relation causes the reference to be passed to the origin, thus deleting the contents in the origin.

Parameters

  • The parameter of data type listrange designates the range which you want to delete. You can also enter several ranges.

  • When you do not enter a parameter, i.e. no range, Plant Simulation deletes the contents of all cells.

For TableFiles entering no parameters means TableFile.delete({2,2}..{*,*}). Plant Simulation does not delete the column index or the row index, no matter if it belongs to the contents or to the format.

If you want to delete the user-defined index row index or column index of TableFiles, you have to do so explicitly by entering row 0 or column 0 respectively.

Example:

 
MyStackFile.delete -- deletes the data range
MyQueueFile.delete({2}..{4})
MyCardFile.delete({1}..{5}, {8})
MyTableFile.delete({2,2}..{*,*})
MyTableFile.delete({0,1}..{0,*}) -- deletes the row
indexMyTableFile.delete({1,0}..{*,0}) -- deletes the column index
