readExcelFile
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Importing and Exporting Data in Text Format > readExcelFile

Syntax:

 
Path.readExcelFile(FileName:string [,Sheet:string, NoDebugger:boolean, Password:string]) → boolean

The method readExcelFile imports data from the designated Excel file (*.xls) into the list or table designated by Path. Note that Plant Simulation overwrites any existing data.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

Plant Simulation uses Excel as COM-server when reading and writing MS Excel-files. For this reason MS Excel has to be installed on your computer. To continue using the previous, no longer supported Excel interface, you can use the start option -NativeExcel.

Parameters

  • The parameter FileName of data type string designates the path to and the name of the Excel file (*.xls).

  • The optional parameter Sheet of data type string designates the worksheet. When you do not enter it and when the Excel file has more than one worksheets, Plant Simulation opens a dialog allowing you to select the desired worksheet.

  • The optional parameter NoDebugger of data type boolean sets if the Debugger opens after an unsuccessful access or not.

    When you do not specify the optional parameter of data type boolean, the Debugger always opens.

    When you enter true, the Debugger does not open and the method returns the value false when an error occurs.

  • The optional parameter Password of data type string designates the password.

Return Value

The return value has the data type boolean.

Example:

 
MyCardFile.readExcelFile("C:\users\johnE\times.xls")
MyTableFile.readExcelFile("C:\temp\product_plan,xls","plant xy anytown")
if MyStackFile.readExcelFile("C:\temp\data.xls", true) = false 
   -- could not read file
end
