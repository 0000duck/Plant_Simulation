User-defined Names
SimTalk Reference > General SimTalk Functions > Names > User-defined Names

Tecnomatix Plant Simulation automatically assigns valid names to objects. Within control methods you yourself have to select names for variables. Here these restrictions apply:

  • The name cannot start with a digit. It can start with a letter or with an underscore (_). This can be followed by letters, digits or the underscore (_), for example MyStation, MyStation1, My_Station_1. The name of an object cannot start with a digit, i.e., you cannot enter 1Station. You cannot use special characters for names.

  • Names are not case-sensitive, Plant Simulation does not distinguish between upper, UpPER and UPPER.

    Data you enter is case-sensitive. When comparing string values Plant Simulation takes upper- and lower-casing into account.

  • The name has to be unique, i.e., it cannot be the same as the name of a keyword, or the name of a built-in functions or a built-in method.

Enter names identifying the role of an object or variable, whenever this is possible. forklift.destination := .building1.station_1 is more meaningful than MU1.attrib := .Frame1.Station21.
