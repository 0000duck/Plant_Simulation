Insert a List into a TableFile from a Frame
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Creating Lists within Lists and Tables > Creating a List within a TableFile > Insert a List into a TableFile from a Frame

When you insert a table from the Class Library into your simulation model it will be Empty as it is unknown at this time how many columns and rows it will contain.

Only when Plant Simulation creates the table, by executing the method createNestedList (StackFile and QueueFile), createNestedList (CardFile), createNestedList (TableFile), it will actually create it and enter a pointer to this new list into the TableFile.

Syntax:

 
Path.createNestedList(index1,index2)
 
The expression Path is the name of the table where Plant Simulation creates the sub-table designated by the parameter [index1,index2]. The method create allows accessing every position. When you create a list in an entry that already contains an entry, Plant Simulation overwrites the existing entry.
