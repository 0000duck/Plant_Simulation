Executing Simulation Studies
2D Reference Help > Tools > ExperimentManager > How It Works and What You Can Do with It > Executing Simulation Studies

In a simulation study we investigate how model parameters, which are a set of input values, influence certain resulting values, which are output values. An experiment consists of one or several simulation runs for which the input values do not change.

When your simulation model only contains deterministic processes, the output values are uniquely determined by the input values. In such a deterministic simulation study an experiment consists of a single simulation run.

When your model contains components, which show an unpredictable behavior, you will use probability distributions for modeling them. Such components encompass, for example, failures of machines. The stochastic processes lead to fluctuating output values. The output values can then be interpreted as realizations of a random number. The objective of an experiment of a stochastic simulation study is to determine the properties of the probability distributions of the output values, such as the mean value and the standard deviation. Within an experiment the ExperimentManager executes several simulation runs with unchanged input values and determines the output values. For this reason we call the simulation runs of an experiment observations. For carrying out the observations, the ExperimentManager changes the seed values of the random number streams which are associated with the stochastic processes. Enter the number of observations for each experiment into the text box Observations per experiment. Compare Modeling Random Processes in the Step-by-Step Help.
