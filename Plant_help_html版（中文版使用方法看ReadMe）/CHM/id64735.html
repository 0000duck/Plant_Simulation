& (SimTalk 2.0) ref (SimTalk 1.0)
SimTalk Reference > General SimTalk Functions > Operators and Expressions > Assignment Operators > & (SimTalk 2.0) ref (SimTalk 1.0)

Before you can to access attributes or methods of objects of type Variable or Method , keep this in mind:

  • When accessing a global Variable Plant Simulation returns its contents and therefore you would send the attribute name or method name to the content and not to the Variable object itself.

  • When accessing a Method Plant Simulation starts executing the method and therefore you would send the attribute name or method name to the result and not to the Method object itself.

If you want to access attributes or methods of one of these objects, you have to use the reference operator . It prevents accessing the contents of the variable or calling the method and returns the reference as an absolute path of data type object. You can then enter this reference or the path in lists or tables and enter them as parameter into other methods.

Example:

 
-- SimTalk 2.0 notation 
var obj: object
obj := &MyMethod
obj := &MyVariable

Example:

 
-- SimTalk 1.0 notation
is
   obj : object;
do
   obj := ref(MyMethod);
   obj := ref(MyVariable);
end;

Place parenthesis around the reference to access the contents of a global variable of data type object. If the reference designates a Method, Plant Simulation is going to call this Method. During the call parameters may be passed.

Example:

 
-- SimTalk 2.0 notation 
MyVariable := .MaterialFlow.Station  -- variable of data type object
var obj := &MyVariable
print obj.Name                          -- returns MyVariable
print obj.Value.Name                    -- returns Station
obj := &MyMethod                        -- reference
obj.execute                             -- call
obj := &methodWithArgument
obj.execute(1.0, true,"parameter")      -- call with parameter

Example:

 
-- SimTalk 1.0 notation
is
   obj : object;
do
   .variable := .MaterialFlow.Station;  -- variable of data type object
   obj := ref(.myVariable);
   print obj.Name;                         -- returns .myVariable
   print (obj).Name;                       -- returns Station
   obj := ref(myMethod);                   -- reference
   (obj);                                  -- call
   obj := ref(methodWithArgument);
   (obj)(1.0, true,"parameter");           -- call with parameter
end;

When using a user-defined attribute of data type method, and you want to access the attributes and methods of that user-defined attribute, then you may not use the reference operator.

Example:

 
Station.method               -- executes the user-defined method
Station.method.methcall(60)  -- schedules the user-defined method 
                                 -- to be called in 60 seconds
