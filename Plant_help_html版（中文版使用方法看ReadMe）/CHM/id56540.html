inOrder
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for the Order of Cells within Lists and Tables > inOrder

Syntax:

 
Path.inOrder(columnIndex:any, Value:any, SortOrder:any)

The method inOrder inserts a new row with the value you want to sort into a sorted CardFile, TableFile or TimeSequence at the position it should hold within that order.

Plant Simulation sorts empty cells at the end of the column for ascending sorting; it sorts them are sorted at the start of the column for descending sorting.

Parameters

  • The parameter any designates the column index.

  • The parameter any designates the value, which you want to sort.

  • The parameter any designates the sort order. Enter the sort direction ascending (up) or descending (down). When you do not enter a sort direction, by entering an empty string (""), the decision over the position at which the entry is to be sorted in is shifted to the next sort direction. When you do not enter a command or a column of a table, the corresponding cell remains empty. The parameters consist of the column index, the value you want to sort in and direction. Separate the parameters with commas.

Suppose that we want to sort a new row containing the values 4711,1,9 into the following table.

The value of the third column, sorted in ascending order (3,9,up) is to be sorted first. Column 1 is not sorted (1,4711,""), column 2 is sorted in descending order (2,1,down). Enter this command to accomplish your goal:

Example:

 
MyTableFile.inOrder(3,9,"up",1,4711,"",2,1,"down")
