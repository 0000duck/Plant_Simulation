Inherit Contents
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Window of Lists and Tables > List Ribbon Tab > Inherit Contents

The command Inherit Contents activates or deactivates inheritance of the contents of the list.

When you insert a list into a Frame, it automatically inherits its contents from its class. You can only change the contents of the new, instantiated list after you deactivate Inherit Contents. Deactivating this command also deactivates Inherit Format. Entering data into a list or table automatically deactivates Inherit Contents.

Plant Simulation does not deactivate Inherit Contents, when you instantiate the class of a list containing a sub-list, and then write to the instance with a Method, or when you write to the sub-list with a Method. If you model like that, you have to deactivate Inherit Contents yourself.
