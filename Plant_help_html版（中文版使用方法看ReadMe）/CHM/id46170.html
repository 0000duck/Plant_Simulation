Stopped
2D Reference Help > Mobile Objects > Shared Properties of the Mobile Objects > Attributes of All Mobile Objects > Stopped

Example:

 
MU-path.Stopped

The attribute Stopped stops (true) the MU designated by MU-path or makes it continue on its way (false) it continue on its way on the Conveyor, the Track, or the TwoLaneTrack.

You can use it, for example, to make an MU, which you want to load onto a pallet, stop and wait until the pallet arrives. Once the pallet has arrived, you can then set the attribute Stopped to false, so that the MU can be loaded onto the pallet. Most often you will use the attribute with a driving control of the Transporter.

When you set the attribute Stopped to true, Plant Simulation stops a possible driving towards the target position. Plant Simulation does not call the Destination control and sets the value of the attribute TargetPosition to the default value -1.

Data Type of the Value You Can Assign

You can assign a value of data type boolean.

Example:

 
param SensorID: integer Front: boolean
@.Stopped := true
wait z_uniform(1,25,35)
@.Stopped := false

Example:

 
.MUs.Part:2.Stopped := true -- MU stops
.MUs.Part:2.Stopped := false -- MU continues
