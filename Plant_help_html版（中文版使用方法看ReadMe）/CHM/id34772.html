Configuring DCOM on the Remote Computer
2D Reference Help > Tools > ExperimentManager > Configuring DCOM for Remote Access > Configuring DCOM on the Remote Computer

First, select general DCOM settings:

  1. On the remote computer, click Start > Control Panel > Administrative Tools > Component Services.

  2. Expand Component Services, expand Computers, and right-click My Computer. Select Properties.

  3. Click the tab Default Properties. Select Enable Distributed COM on this computer. Set the Default Authentication Level to Connect (None also works). Set the Default Impersonation Level to Identify (Impersonate also works).

  4. Click the tab Default COM Security.

  5. Under Access Permissions click Edit Default. Add SYSTEM, INTERACTIVE, and NETWORK. You also have to include the user whose authentication credentials will be used to access the COM application to this list. You can, for example, add the specific user or simply add a group the user belongs to. You might enter:

    • Domain\Username (A specific user)

    • Domain\Administrators (All administrators on a specific domain)

    • Everyone (All users)

  6. Under Launch Permissions click Edit Default. Make sure that the Launch Permissions have the same values as the Access Permissions.

  7. Click the tab Default Protocols. Make sure Connection-oriented TCP/IP is listed first. Click OK.

Then, configure the COM application you would like to access:

  1. Expand Component Services, expand Computers, expand My Computer, and click DCOM Config. Right-click the application you would like to configure and select Properties. To configure Plant Simulation, select Tecnomatix Plant Simulation 13.1.

  2. Click the tab General. Set the Authentication Level to Default.

  3. Click the tab Location. Select Run application on this computer.

  4. Click the tab Security. Set Launch Permissions to Customize. Set Access Permissions to Customize. Set Configuration Permissions to Customize.

  5. Click Edit and add yourself to the list of users.

  6. Click the tab Identity. Select The launching user.

  7. Click the tab Endpoints. Select default system protocols.

  8. Click OK.

When no identical user account exists on both computers, no authentification can take place. To deactivate authentification, you can create the following registry key of type DWORD and assign it the value 1: HKEY_CURRENT_USER/Software/Tecnomatix/Tecnomatix Plant Simulation 13.1/Settings/ RPC_C_AUTHN_LEVEL_NONE. Please note that this might pose a security hole.
