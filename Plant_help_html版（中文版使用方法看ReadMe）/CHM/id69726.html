Forward Blocking List
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Dialog Items of the Objects > View Menu > Forward Blocking List

The menu command Forward Blocking List opens a table showing all MUs that wanted to move onto the object at its entrance, i.e., moving in the forward direction from the predecessor, but could not. When an MU cannot be moved on, Plant Simulation stops it and enters it into the forward locking list.

The table shows the name of the MU (object) and the time at which the MUattempted to leave the preceding object (time).

The predecessor is the object that is connected to the selected object with a Connector and that precedes it in the sequence of stations in the simulation model.

If you enter a certain position on the target object using the parameter of data type length, the MU does not enter itself into the Forward/Backward Blocking List as Plant Simulation cannot watch/observe at which point in time this position is available again.

In addition Plant Simulation does not enter an MU into the Forward/Backward Blocking List when it is located on a moving Transporter.

Plant Simulation removes an MU, which is located on the loading space of a Container/Transporter, from the Forward/Backward Blocking List when the Container/Transporter itself is moved.
