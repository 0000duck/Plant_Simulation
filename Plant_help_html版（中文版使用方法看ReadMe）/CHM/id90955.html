Executing Distributed Simulations
2D Reference Help > Tools > ExperimentManager > Advanced Settings > Executing Distributed Simulations

If your computer has a multi-core processor, you can significantly shorten the time it takes to run a study by executing the simulation runs in parallel. The ExperimentManager recognizes the number of cores and starts several Plant Simulation processes on your computer. You can configure distributed simulations under Tools > Advanced Settings > Distribution.

As for a distributed simulation, without using rule-based settings, the simulations for all experiments can be executed in parallel, the ExperimentManager then only shows the number of simulations that have already been executed in its dialog.

If you have several Plant Simulation licenses of the same type and if you would like to run time-consuming and extensive simulation studies, the ExperimentManager can execute the simulation runs on several computers within your local network (LAN).

You can also execute an optimization with the GAWizard using the distributed simulation. It provides the same functionalities as the ExperimentManager.

Executing the simulation runs in parallel on several computers can significantly reduce the time the simulation study takes. The ExperimentManager on the control computer starts Plant Simulation processes on other computers, on the so-called simulation machines , loads the model, which was saved last, on the simulation machines and manages the simulation results, which the individual simulation machines arrived at. You will define the experiments in the ExperimentManager on the control computer.

If you want to run distributed simulations, you have to configure the control computer and all other computers to enable them to do so, compare Configuring DCOM for Remote Access.

The entire simulation model is transmitted to the simulation computers within your LAN when you start an experiment study. If this process takes a very long time, we recommend to not employ distributed simulations. Be aware that the simulation model should not access external files.

If a multi-processor computer is available for distributed simulations, you can use this computer multiple times as your simulation computer. The ExperimentManager recognizes a computer with several cores. Select the check boxes Distributed simulation in both dialogs as shown below to use distributed simulations. If you did not enter any simulation computers yet and you then select the check box, the ExperimentManager executes distributed simulations according to the number of cores. In this case the table of the simulation computers shows localhost(n). Then number n, designating the number of cores, does not show the actual number of used cores though. This ensures that distributed simulations with this model on another computer uses then number of available cores.

In this case the simulation machines are not visible. When the attribute RemoteMachinesVisible of the ExperimentManager has the value true, the simulation machines will be visible. You can view and change the attribute in the dialog Show Attributes and Methods.

Click Insert and Remove to designate specific computers within your LAN as simulation machines. Instead, you can also double-click an entry in the list. Click Test to check if the selected remote computer is able to run a remote simulation or not.

After the simulation study is finished, you can track the execution of the simulation runs for all simulation machines. When errors occurred on the simulation machine, you can analyze the cause of the error by clicking the button Protocol. This opens the table MachinesTable containing information about all used simulation machines. Column 8 of this table recorded the communication of the controlling computers with the respective simulation machine.

At the start of the simulation study, the ExperimentManager starts Plant Simulation processes on all simulation machines with the license you selected under Remote license, compare Execute Experiments with the ExperimentManager. These Plant Simulation processes execute simulation runs one after the other.

Enter the CPU time into the text box Terminate after after which the ExperimentManager terminates a simulation run on a simulation machine.

When no additional simulation runs have to be executed for the simulation study, all Plant Simulation-processes on the simulation machines will be terminated.

To start the Plant Simulation process on the simulation machine with a lower priority, select the check box Lower priority. Then you can work better on the simulation machine at the same time.

After the simulation study has been finished, you can open an overview over the simulation machines, which the simulation experiment used, by selecting Evaluate Simulation Machines on the context menu. Analyzing the number of simulation runs on the simulation machines and their mean run times enables you can to better organize future distributed simulation studies. For a small number of parallelizable simulation runs it might be better to not use a slow simulation machine, as this computer might cause faster computers to wait for it.

Evaluating the simulation machines

During the simulation study you can observe how parallelizable simulation runs are executed. Select Show Distributed Simulation on the context menu to do so.

When employing distributed simulation, you cannot handle data within your LAN the way you are used to do. We recommend to not read data from files, but to first save them to your model. Note that you cannot save the active model while distributed simulations are running.

When you would like to log special events or system states in the Report of the ExperimentManager, the Frame containing the simulation model has to contain corresponding Methods with prompt messages.

You can designate the number of the simulation run, which belongs to the row in the JobsTable , on the associated simulation computer with the attribute currentTaskNo of the ExperimentManager. 

promptMessage(to_str("Emptywarehouseinsimulationrun",Experimentmanager.currentTaskNo,"at time",EventController.SimTime))

When you use rules, all observations for an experiment have to be finished before the next experiment can start. For this reason only the observations of a single experiment can be executed in parallel when you employ distributed simulation.
