Assigning a Value to a Variable
2D Reference Help > Information Flow Objects > Variable > Methods of the Variable > Assigning a Value to a Variable

Syntax:

 
Path := any

When assigning a new value to the Variable, this value has to be compatible with the active data type. A value is compatible if Plant Simulation can automatically transform it into a correct value for the task. A change of value may occur because:

  • Plant Simulation cannot display digits after the decimal point and cuts them off. It transforms, for example, the real value 3.9 into the integer value 3.

  • Loss of significance occurs for great value differences. Floating point operations only save a certain number of digits and the magnitude of a number (mantissa and exponent). Plant Simulation deletes additional digits created during a calculation and the number does not return the expected value. This kind of error cannot be avoided and only appears if the difference in size is at least 107.

When you assign a Variable to a Method keep in mind that Plant Simulation calls and executes this Method, when you only enter the nameof a Method. This way, the Method is returned, instead of to the reference you would like to receive. Use the reference operator, to get the reference!

Example:

 
Variable := Value
Variable := &MyMethod
