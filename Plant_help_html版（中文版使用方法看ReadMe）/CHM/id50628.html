outIn
2D Reference Help > Mobile Objects > Shared Properties of the Mobile Objects > Methods of All Mobile Objects > Methods for the State of the Mobile Object > outIn

Syntax:

 
MU-path.outIn(SecondsTillOut:time[, WorkOnFinishedPartAgain:boolean:=false])

The method outIn causes the MU designated by MU-path to exit the object on which it is located after the time you specify.

Parameter

  • The parameter SecondsTillOut of data type time designates the number of seconds. Plant Simulation creates an Out event for the MU in the number of seconds starting at the current point in time of the simulation run. The time you enter has to be equal to or greater than 0. For a fully processed part you can also enter -1 to start processing again. Here the parameter WorkOnFinishedPartAgain has to have the value true.

    On length-oriented material flow objects you can only use the method for MUs, which are ready to exit.

  • The optional parameter WorkOnFinishedPartAgain of data type boolean sets if an already processed part will be processed again on point-oriented objects. When the parameter has the value true or the part has not been processed all the way yet, the time counts as processing for statistical purposes. When the parameter has the value false and the part has already been processed all the way, the time counts as waiting for statistical purposes.

    For parts that are processed all the way and for the setting WorkOnFinishedPartAgain(true), you can also specify -1 for the parameter SecondsTillOut. The value -1 signifies that Plant Simulation determines the processing time according to the attribute ProcTime of the material flow object, meaning that it either determines a random number or that it sets a constant processing time.

    This setting is particularly useful when an MU is created on a station. Then it is full processed and ready to exit. If it is to be processed still, you can enter outIn(-1, true) directly after create to start processing.

Example:

 
Station.Cont.outIn(10)

Example:

 
var p : object
p := .MUs.part.create(Station)
p.outin(-1, true)
