_3D.groupGraphics
SimTalk Reference > 3D-related SimTalk Functions > Built-in Methods for Detailed Access To Graphics > _3D.groupGraphics

Syntax:

 
Path._3D.groupGraphics(GraphicPath:any[], Children:any[][,GraphicGroupName:string]) → integer

The method _3D.groupGraphics combines the designated graphics in the Frame designated by Path into a group. You can specify the graphics to be grouped either by their numeric indexes or with their names.

The method deactivates graphic inheritance.

Parameters

  • The parameter GraphicPath is an array of data type any which sets the path to the shared parent node of the graphics that are to be grouped. The parameter is described under The Parameters GraphicPath and GraphicGroupName.

    Here you can also enter an empty path, i.e., the expression [] to group graphics which are located directly within a graphic group.

  • The optional parameter GraphicGroupName of data type string designates the name of the graphic group you are working with. The parameter is described under The Parameters GraphicPath and GraphicGroupName.

Return Value

The return value has the data type integer and specifies the numerical index of the group, compare _3D.getGraphicIndex.

Example:

 
MyFrame._3D.groupGraphics([1],[2,3])
 -- groups the graphics at the positions [1,2] and [1,3]

Example:

 
MyFrame._3D.groupGraphics([1,2,4],[2,3], "deco")
 -- groups the graphics at the positions [1,2,4,2] and [1,2,4,3]
