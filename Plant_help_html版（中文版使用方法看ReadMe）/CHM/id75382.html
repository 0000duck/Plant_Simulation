readLn
2D Reference Help > Information Flow Objects > FileInterface > Methods of the FileInterface > readLn

Syntax:

 
Path.readLn → string

The method readLn reads the line, in which the file cursor is located, into the file and increases the counter by one. It passes the entire line as a string. You may have to employ one of the conversion functions, such as str_to_num, to convert the string into another data type.

If the file is not open for write, the FileInterface opens it and reads the line. When the file cannot be opened, for example because the path to the file is invalid, the function fails with an error. To avoid this error, you can explicitly call the method open, which returns false if the file cannot be opened.

Return Value

The return value has the data type string.

Example:

 
-- A file contains numbers in several lines. These will be read 
-- into the second column of a TableFile
var y: integer, line: string
y := 1
while not MyFileInterface.EoF  -- until end of file reached
   line := MyFileInterface.readLn -- get next line
   MyTableFile[2,y] := line
   y += 1
end
