asObject
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Indirectly Accessing Lists and Tables > asObject

Syntax:

 
Path.asObject(Column:any, Row:any) → any

The method asObject can be applied to cells of lists or tables, which have the data types string or object. When the table contains a valid path to an object, the method returns this object.

Parameters

  • The parameter Column of data type any designates the column of the table.

  • The parameter Row of data type any designates the row.

Return Value

The return value has the data type any.

When the table cell, which the method asString returned, contains a valid path to a track of a TwoLaneTrack, the method returns this lane. This returned value does not have the data type object and thus cannot be assigned to a variable of data type object. You can, however, assign it to a local variable of data type any. When a table cell of data type object contains a path to a lane, reading the contents of the cell with brackets (table[x,y]) returns VOID as a lane does not have the data type object.

Example:

 
var a : any
if .MUs.Transporter:1.getRouteToDestination(TableFile) 
a := MyTableFile.asObject(1,1)
if isObject(a) 
print "first object: ", a.name
else
print "first object is a two-lane track: ", a.~.name
end
print "the length is: ", a.length
