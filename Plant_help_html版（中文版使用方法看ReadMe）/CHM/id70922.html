_3D.TypeOfAnimations.createAnimationLine
SimTalk Reference > 3D-related SimTalk Functions > Built-in Methods and Read-Only Attributes for Animations in 3D > _3D.TypeOfAnimations.createAnimationLine

Syntax:

 
Path._3D.MUAnimations.createAnimationLine(Name:string) → boolean
Path._3D.SelfAnimations.createAnimationLine(Name:string) → boolean
Path._3D.CameraAnimations.createAnimationLine(Name:string) → boolean

The method _3D.TypeOfAnimations.createAnimationLine creates a new saved animation path of type Lines.

You can set the values of the animation path with the method _3D.TypeOfAnimations.setTable.

Parameter

The parameter Name of data type string designates the name of the new animation path to be created.

Return Value

The return value has the data type boolean.

It tells if the animation path was created successfully, i.e., if the name has not been assigned yet.

Example:

 
var mua := Buffer._3D.MUAnimations
var created : boolean := mua.createAnimationLine("Animation2")
if created -- The animation line was created. 
   var t: table -- Copy the ‘Default’ line values to the table.
   mua.Default.getTable(t)
   mua.Animation2.setTable(t)
end
