Stop On
2D Reference Help > Information Flow Objects > Method > The Method Window > Method Debugger > Menu Bar of the Method Debugger > Debug Menu > Stop On

The menu command Stop On opens a submenu on which you can select at which event the Debugger stops when executing a Method.

Stop on Controls

The menu command Stop on Controls stops the simulation when the EventController encounters a Method--an entrance or exit control you entered into an object--that is started from an object and before executing the calling Method. Plant Simulation only executes calls by control methods during a simulation run.

Instead, you can also click Stop on Controls on the Debugger Ribbon Tab.

Stop on Formulas

The menu command Stop on Formulas stops the simulation and opens the Debugger, whenever a formula is evaluated.

Instead, you can also click Stop on Formulas on the Debugger Ribbon Tab.

Stop on Error Handler

The menu command Stop on Error Handler stops the simulation and opens the Debugger whenever an error handling method is executed.

Instead, you can also click Stop on Error Handlers on the Debugger Ribbon Tab.

Stop on Wakeup

The menu command Stop on Wakeup stops the simulation and opens the Debugger whenever the execution of a suspended method is being continued.

Instead, you can also click Stop on Wakeup on the Debugger Ribbon Tab.
