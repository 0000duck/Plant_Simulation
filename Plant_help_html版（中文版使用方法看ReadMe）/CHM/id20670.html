Check How Many Parts Were Introduced into the Plant
Step-by-Step Help > Animating the Simulation Model and Viewing the Results > Viewing and Visualizing Statistics > Viewing Statistics in the Dialogs of the Objects > Check How Many Parts Were Introduced into the Plant

Next to the values described under Viewing Statistics in the Dialogs of the Objects, you can make the Source record which MUs it actually created at which point in time.

  • To write all events for which the Source produced MUs during a simulation run to a table, select Creation table on the tab Statistics.

    The Source records these events in addition to the resource statistics it collects in any case.

  • To open the creation table, click Open.

    Each row on the table shows an event for which the Source produced an MU:

    • The column Name show the name of the produced MU class.

    • The column Path show the path of the MU class, the name of the MU and its number.

    • The column Time of Generation show the time when the Source produced the individual MU.
