_3D.resetAnimationTime
SimTalk Reference > 3D-related SimTalk Functions > Built-in Attributes and Methods for MU Animations > _3D.resetAnimationTime

Syntax:

 
Path._3D.resetAnimationTime

The method _3D.resetAnimationTime sets the animation time of an MU on the object, on which it is located to 0. This affects time-dependent animations, i.e., MU animations , which are playing on an animation point in 2D but are playing on a path that is longer than 0 in 3D. A time-controlled MU animation always starts when the MU moves onto the object on which it will be processed or transported.

You can use the method resetAnimationTime to start an animation from 0 at a later point in time. If you do not do this, the animation to be used will be changed, but does not start at 0, but with an initial time offset, which corresponds to the time which the MU is already located on the current object.

Example:

 
@._3D.resetAnimationTime
