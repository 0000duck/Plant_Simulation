writeXMLFile
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Importing and Exporting Data in Text Format > writeXMLFile

Syntax:

 
Path.writeXMLFile(FileName:string [,NoDebugger:boolean]) → boolean

The method writeXMLFile exports the contents of the list or table to the designated XML file. Note that Plant Simulation overwrites any data that this file might already contain.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

Parameters

  • The parameter FileName of data type string designates the name of the XML file into which Plant Simulation exports the data.

  • The optional parameter NoDebugger of data type boolean sets if the Debugger opens after an unsuccessful access or not. When you do not enter a value for the optional parameter of data type boolean, the Debugger always opens. When you enter true the Debugger does not open and the methodreturns the value false when an error occurs.

Return Value

The return value has the data type boolean.

Example:

 
tablefile1.writeXMLFile("D:\MyName\Public\xmltest.xml")
