find
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for the Order of Cells within Lists and Tables > find

Syntax:

 
Path.find(Value:any) → boolean
Path.find(Range:listrange, Value:any) → boolean

The method find searches for a value in the list/table designated by Path. Plant Simulation searches in the list/table starting at the cursor position which you can set using setCursor and then continues column-wise until it finds the value. When it finds a value, the method returns true and sets the cursor into that cell. It returns false when it did not find the value. If several values exist, Plant Simulation sets the cursor into the first one it finds. A new search starts at the position after the last result and returns the next entry towards the end of the list/table. To begin a new search use setCursor to set a new start position. Plant Simulation searches tables column-wise starting either in the first cell of the first column or in the first cell of the range you specified.

The method find(void) only searches the range of the list that contains entries, i.e., between position 1 and the positions set by the methods XDim or YDim.

When searching for a string, the string comparison is case-insensitive.

Plant Simulation starts searching at the current cursor position. This only applies when the cursor is located within one of the specified ranges. If this is not the case, Plant Simulation starts searching within the first specified range and then searches within the following ranges if it does not find the value, etc.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

Parameters

  • The parameter Value of data type any designates the value you want to find.

  • The parameter Range of data type listrange designates the range between which Plant Simulation searches to the position of the cursor, for tables the position of both cursors.

    When the cursor is located within one of the specified ranges, Plant Simulation does not search the preceding ranges, not even after the search was negative for the other ranges. When the cursor is located in the middle of the range, Plant Simulation only searches the range after the cursor, but not the range before it. When you do not enter a range, Plant Simulation searches the list starting at the position of the cursor to the end of the list.

When a search was concluded without finding the value, the internal cursor afterwards points to the starting position from which the search was started.

Example:

 
MyStackFile.find(12.34) -- finds the floating point value 12.34
MyQueueFile.find(42) -- finds the integer value 42
MyCardFile.find({1},{4}..{8},"drill")
--finds the string "drill" in row 1 and in rows 4 to 8
MyTableFile.find("a") 
-- finds the string "a" in the entire table
MyTableFile.find({3,*},"a") 
-- finds the string "a" in column 3
MyTableFile.find({1,1}..{4,*},"a") 
-- finds the string "a" in columns 1 to 4 in all rows

Example:

 
var MyTableFile.setCursor(2,7) -- searches for abc towards the end of the table
if MyTableFile.find("abc")
   print MyTableFile.CursorX, " ",MyTableFile.CursorY
else
   print "not found"
end

When you enter the parameter void, the method find finds blank cells in the list. Note that cells containing an empty string ("") are not empty!

Return Value

The return value has the data type boolean.

Example:

 
if data.find(void)
   print ("data not complete")
end
