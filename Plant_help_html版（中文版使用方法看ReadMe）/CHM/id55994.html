Export Excel File
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Window of Lists and Tables > List Ribbon Tab > Export Excel File

The command Export Excel File saves the list as a Microsoft Excel worksheet (.xls). It opens the dialog Save As in which you select the drive, and the file type and enter a new name for the file to be saved. Click Save to save the file.

Plant Simulation uses Excel as COM-server when reading and writing MS Excel-files. For this reason MS Excel has to be installed on your computer. To continue using the previous, no longer supported Excel interface, you can use the start option -NativeExcel.

Plant Simulation only exports integers in the range between -536.870.912 and 536.870.911 in the required format. Plant Simulation saves values outside of this range as 0.

Plant Simulation exports the data with the settings that you selected for the units. Suppose that you selected Foot as the length unit for a column of data type Length. Then Plant Simulation exports the values in Feet to Excel and interprets the values as Feet when you import them back to Plant Simulation.

When Plant Simulation imports an Excel worksheet, it attempts to match the individual columns to the possible Plant Simulation data types. This only works when you created the columns of the Excel worksheets of a singular type. When you created a cell of a column with a string for example, the entire column can only have the data type string.

An exception can be row 0. If the table that you are going to import, has a column index, Plant Simulation interprets row 0 as the column index and does not use it for determining the data type of the column.
