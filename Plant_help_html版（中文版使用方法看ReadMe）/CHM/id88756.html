In this example we use a property of the part is a condition for a breakpoint.

Plant Simulation inserts a breakpoint, when a part of the class .MUs.Part, which is less than 100 meters long, causes an Out event on Station1. Note that the unit depends on the settings you selected under File > Model Settings/Preferences > Units > Length.

If you would like Plant Simulation to create a trace file that tracks all events, enter a name into the text box Trace File in the dialog Event Debugger and select the check box Trace active.

If you would to only create a trace file for a certain event, enter a file name into the text box Trace File in the dialog Breakpoint.
