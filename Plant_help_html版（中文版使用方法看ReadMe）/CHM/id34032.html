Failed
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Dialog Items of the Objects > Failed

To fail the object, select this. From then on, the object will not receive any parts. Parts can exit the object though, when you program this in a Method.

The program stops processing and set-up processes until the end of the failure, i.e., it adds the duration of the failure to these times.

When you manually fail a station, it remains Failed while any of the failure profiles you defined is Active. It will change to not failed once the last failure (DisruptionEnd) of the last failure profile ends or when you clear the checkbox .

When a station fails, and a pause takes place during the failure, the failure time of the station will still be consumed, although the station is paused.

Statistics counts the times during which these two states overlap as pause time.

The object is failed when you activate this check box or when you activate one of its failure profiles on the Tab Failures.

Click OK or Apply to accept your changes.

The LED of a failed station is red .

The length-oriented objects do not display LEDs. Plant Simulation shows the border of a failed length-oriented object red .

Plant Simulation resets failures when you click Reset Simulation the simulation model.
