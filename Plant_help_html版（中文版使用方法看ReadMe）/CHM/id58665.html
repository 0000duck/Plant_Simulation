Built-in Methods and Read-Only Attributes for Animations in 3D
SimTalk Reference > 3D-related SimTalk Functions > Built-in Methods and Read-Only Attributes for Animations in 3D > Built-in Methods and Read-Only Attributes for Animations in 3D

Plant Simulation provides a set of functions for each of the following three animation types:

  • All object, which accept MUs or Workers provide MU animations:

  • All material flow objects except for Connectors and Interfaces.

  • Objects , i.e., the objects, which you can address with the method _3D.getObject.

  • Frames

  • All objects provide self animations, except for Basis, Connectors, Interfaces and MUs.

  • Only Frames provide camera animations

You can cache the entirety of all animations of an animation type in a value of data type any. You will, for example, enter: 

var a : any := .MaterialFlow.PickAndPlace._3D.SelfAnimations

The three animation types share a common set of functions of data. They also provide a number of functions specific to a certain type. All of the functions can handle saved animations though.

Plant Simulation provides these attributes and methods for animation paths in 3D:

_3D.TypeOfAnimations.Count, _3D.TypeOfAnimations.createAnimationCurve, _3D.TypeOfAnimations.createAnimationLine, _3D.MUAnimations.createAnimationPoint, _3D.TypeOfAnimations.createAnimationSpline, _3D.TypeOfAnimations.getAnimation, _3D.TypeOfAnimations.AnimationPathName.getTable, _3D.TypeOfAnimations.AnimationPathName.setTable, _3D.TypeOfAnimations.getAnimation, _3D.TypeOfAnimations.getTable, _3D.TypeOfAnimations.setTable, _3D.RobotArmAnimations.getTable, and _3D.RobotArmAnimations.setTable.

The animation types self animations and camera animations have saved and temporary animation paths.

The animation type MU animations only has saved animation paths.
