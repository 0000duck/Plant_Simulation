The cell Variable shows the name of the object that calls the Method. If that is another Method, the cell contains the name of that Method. Otherwise it contains the name of the object in which an MU triggered a control call. You cannot change the contents of the cell!

Press F2 to open the dialog window of the calling object. If a user-defined Method contains the identifier ? and you would like to test the Method without running the simulation, you have to enter a suitable object in the object Method. The Debugger uses the contents of the cell  ? of the Method.

For an in-depth description, compare Anonymous Identifiers.
