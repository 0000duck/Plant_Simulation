writeFile
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Importing and Exporting Data in Text Format > writeFile

Syntax:

 
Path.writeFile(FileName:string [,NoDebugger:boolean, CodePage:string]) → boolean

The method writeFile exports the contents of a list or table to the designated text file. Note that Plant Simulation overwrites any data that this file might already contain.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

Parameters

  • The parameter FileName of data type string designates the name of the text file into which Plant Simulation exports the data.

  • The optional parameter NoDebugger of data type boolean sets if the Debugger opens after an unsuccessful access or not.

    When you do not specify the optional parameter of data type boolean, the Debugger always opens.

    When you enter true, the Debugger does not open and the method returns the value false when an error occurs.

  • The optional parameter CodePage of data type string sets the encoding you want to use: ANSI, UTF-8, or Unicode.

    If you don’t specify this parameter, the encoding UTF-8 is used.

Return Value

The return value has the data type boolean.

Example:

 
MyStackFile.writeFile("C:\users\nelly\times.txt")
MyTableFile.writeFile("C:\temp\run1.txt")
if MyCardFile.writeFile("C:\temp\data.txt", true) 
   -- data written successfully
else -- take appropriate measures
end
