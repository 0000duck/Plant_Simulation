Failed
2D Reference Help > Mobile Objects > Transporter > Attributes of the Transporter > Failed

Syntax:

 
MU-path.Failed

The attribute Failed sets if the Transporter designated by MU-path is Failed (true) or not (false). After the value is changed, Plant Simulation executes the control FailCtrl.

When you manually fail the Transporter, it remains failed while any of the failure profiles you defined is active. It will change to not failed once the last failure (DisruptionEnd) of the last failure profile ends or when you assign false to the attribute.

Data Type of the Value You Can Assign

You can assign a value of data type boolean.

Example:

 
if MyTrack.Cont.Failed
   MyTrack.Cont.Failed := false
end

Plant Simulation resets all failures during the reset phase of the simulation run.
