str_to_time
SimTalk Reference > General SimTalk Functions > Predefined Functions > Functions for Converting Data Types > Functions for Converting Times and Dates > str_to_time

Syntax:

 
str_to_time(Data:string[,FormatLikeDialogs:boolean]) → time

The function str_to_time converts the parameter designated by the parameter of data type string into a time-statement of data type time. Plant Simulation analyzes the statement from right to left. It has this format hh:mm:ss.ss. The value "1:00.1" stands for one minute plus one tenths of a minute.

Conversion depends on the setting you selected under File > Model Settings/Preferences > Units > Time scale.

The statement str_to_time("1:30.50") returns different values, depending on the setting you selected under File > Model Settings/Preferences > Units > Time scale.

Setting

Seconds

Time scale 1/1.0 Transfer if 24:60:60

90.5

Time scale 1/60.0 Transfer if 30:24:60

5430

Time scale 1/0.6 Transfer if 24:60:100

78.3

Parameters

  • The parameter of data type string designates the data to be converted.

  • The optional parameter FormatLikeDialogs of data type boolean sets that the same time format as in the dialogs of the objects will be used (true). When you do not enter a value for the parameter or when you set it to false, Plant Simulation uses the same time format as in tables.

Return Value

The return value has the data type time.

Example:

 
var t: time
t := str_to_time("48:00:00.0")     -- 48 hours
self.methcall(str_to_time("5:00")) -- 5 minutes
