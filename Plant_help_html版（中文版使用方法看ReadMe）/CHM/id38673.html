importExporter
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Methods of the Material Flow Objects > Methods of the Importer > importExporter

Syntax:

 
Path.imp.importExporter(Worker:object/<Workers:array[, MU:object]) → boolean
Path.failImp.importExporter(Worker:object/<Workers:array[, MU:object]) → boolean
Path.setupImp.importExporter(Worker:object/<Workers:array[, MU:object]) → boolean
Path.transportImp.importExporter(Worker:object/<Workers:array[, MU:object]) → boolean

You can use the method importExporter in the Request control of the importer. It returns if the Workers/Exporters were successfully brokered (true) or not (false).

The Broker does not save a failed mediation. You thus have to check beforehand if the Workers can be brokered and you yourself have to start another attempt to broker the Workers.

Parameters

  • The parameter Worker of data type object designates a specific Worker/Exporter.

    The parameter Workers of data type array designates several Workers/Exporters.

  • The optional parameter MU of data type object designates the MU for which the Worker/Exporter was requested.

Return Value

The return value has the data type boolean. It returns if the Workers/Exporters were successfully brokered or not.

Example:

 
MyStation.failImp.importExporter(MyWorker:1)
ParallelStation.imp.importExporter([MyWorker:1, MyWorker:2])
