RootFolder
SimTalk Reference > General SimTalk Functions > Names > Anonymous Identifiers > RootFolder

The anonymous identifier RootFolder designates the folder in the Class Library in which you store Methods, which a number of objects use. This prevents wasting main memory and avoids long paths as Plant Simulation then calls this method once only from the Class Library instead of a large number of instances of this method.

When you use the anonymous identifier RootFolder in a Method, the RootFolder designates a folder in the Class Library for which you have set the attribute RootFolder . In this case Plant Simulation searches for such a folder, starting with the class of the Method, upwards in the hierarchy of objects.

We might, for example, set the attribute RootFolder to true for the folder InformationFlow. We then duplicate the object Method in the folder InformationFlow. Plant Simulation creates the class Method1. When we then insert the Method object called Method into a Frame and enter rootfolder.Method1 as source code, Plant Simulation calls Method1 from the Class Library. This also is the case when we insert this Frame into another Frame.

You can also use the anonymous identifier RootFolder within control methods. Here Plant Simulation looks for the folder for which you set the attribute RootFolder, starting with the class of the Frame into which you inserted the object in which you programmed the control.
