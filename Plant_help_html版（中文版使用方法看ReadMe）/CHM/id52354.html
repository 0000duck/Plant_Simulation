Fail control
2D Reference Help > Mobile Objects > Transporter > Dialog Window of the Transporter > The Tools Menu > Fail control

Click and select a Method object in the dialog Select Object. Enter the source code of the fail control. The Transporter calls it, whenever its Failed state changes. To accomplish this, either select the check box Failed or assign true to the attribute Failed.

The standard fail control looks like this:

Within the control, you can access the object calling the control using the anonymous identifier ?.

The program passes two optional parameters to the Method that is called:

  • The optional parameter FailureIsStarting of data type boolean indicates if the failure starts (true) or ends (false).

  • The optional parameter ProfileNameof data type string indicates the name of the failure profile that caused the failure.

    Provided the fail control declares both parameters, it will not only be called when the object changes to the failed-state or when the failed-state ends, but also each time when a failure profile is Failed or when the failure is removed.

    Let’s say failure profile1 is already failed and then you also fail failure profile2 in addition, then the fail control will still be called although the object remains in the failed-state.

    To open the dialog of the object, whose name you entered into the text box, press the F2 key.
