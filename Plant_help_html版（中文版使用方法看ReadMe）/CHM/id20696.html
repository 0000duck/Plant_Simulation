writeExcelFile
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Importing and Exporting Data in Text Format > writeExcelFile

Syntax:

 
Path.writeExcelFile(FileName:string [,Sheet:string, NoDebugger:boolean=false]) → boolean

The method writeExcelFile exports the content of a list or table to the designated Excel file. Plant Simulation overwrites any existing content of the file.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

Plant Simulation uses Excel as COM-server when reading and writing MS Excel-files. For this reason MS Excel has to be installed on your computer. To continue using the previous, no longer supported Excel interface, you can use the start option -NativeExcel.

Parameters

  • The parameter FileName of data type string designates the Excel file into which Plant Simulation exports the data.

    The file name has to have one of the following extensions: .xls, .xlsx, .xlsm, or . xlsb. The file extension determines which Excel file format Plant Simulation writes.

  • The optional parameter Sheet of data type string designates the worksheet. When you do not enter it, Plant Simulation saves the list with the name of the worksheet, as Table1.

  • The optional parameter NoDebugger of data type boolean sets if the Debugger opens after an unsuccessful access or not.

    When you do not specify the optional parameter, the Debugger always opens.

    When you enter true, the Debugger does not open and the method returns the value false when an error occurs.

    The default value is false.

Return Value

The return value has the data type boolean.

Example:

 
MyStackFile.writeExcelFile("C:\users\nelly\factory_a.xls")
MyTableFile.writeExcelFile("C:\temp\factory_a.xls","Engines",true)
if MyCardFile.writeExcelFile("C:\temp\data.xls") 
   -- data written successfully
else -- take appropriate measures
end
