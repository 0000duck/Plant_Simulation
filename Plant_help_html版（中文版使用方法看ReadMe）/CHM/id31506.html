Standard Transfer Behavior
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Standard Transfer Behavior

If an object has more than one successor, Plant Simulation cyclically moves the MUs on to each of the succeeding stations in turn. If one of the successors is full, failed or paused, then it does not move any MUs to this station until it is unblocked and ready to receive parts again.

You can change the standard transfer behavior on the Tab Exit of the material flow objects.
