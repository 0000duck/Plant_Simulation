_3D.importGraphics
SimTalk Reference > 3D-related SimTalk Functions > Built-in Attributes and Methods for Accessing Graphics > _3D.importGraphics

Syntax:

 
Path._3D.importGraphics(BasePoint:real[3], FilePath:string, YUpToZUp:boolean, MoveContentToZeroY:boolean [, GraphicGroupName:string]) → integer

The method _3D.importGraphics imports a graphic from the file designated by Path and adds it to the existing graphic.

3D can import files of these types:

3D uses the file name of the graphic as the name of the graphic node.

Parameters

  • The parameter BasePoint of data type array of data type real which sets the base point at which the graphic will be inserted.

  • The parameter FilePath of data type string sets path to the graphic to be imported.

  • The parameter YUpToZUp of data type boolean sets if up-direction, i.e., the y-direction of the imported graphic, will be converted from the y-direction to the z-direction.

  • The parameter MoveContentToZero of data type boolean sets if the visible center of the imported graphic is to be moved to the center of the scene defined by the coordinates 0,0,0 (true) or not (false).

  • The optional parameter GraphicGroupName of data type string designates the name of the graphic group you are working with. The parameter is described under The Parameters GraphicPath and GraphicGroupName.

Return Value

The return value has the data type integer and specifies the numerical index of the imported graphic, compare _3D.getGraphicIndex.

Example:

 
var i := MyFrame._3D.importGraphics([0,0,0], "C:\Program Files\Siemens\Tecnomatix Plant Simulation 14.0\3D\jt-graphics\RobotKuka.jt", false, true)
MyFrame._3D.setGraphicName([i])

Example:

 
var i := MyFrame._3D.importGraphics([0,0,0], "C:\Program Files\Siemens\Tecnomatix Plant Simulation 14.0\3D\jt-graphics\RobotKuka.jt", false, true], "deco") 
MyFrame._3D.setGraphicName([i], "RobotKuka")
