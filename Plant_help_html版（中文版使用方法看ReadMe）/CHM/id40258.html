Step Into
2D Reference Help > Information Flow Objects > Method > The Method Window > Method Debugger > Menu Bar of the Method Debugger > Debug Menu > Step Into

The menu command Step Into opens a submenu on which you select into which parts of the source code the Debugger steps when debugging a Method.

Step into Controls

The menu command Step into Controls steps into other call chains to debug the executed Method. Such a call chain can either be a control or a woken up suspended Method, for example could the instruction @.move activate an entrance control.

Step into Formulas

The menu command Step into Formulas steps into other formulas to debug the executed Method. For example could the instruction print table[1,1] compute a formula, which is located in this cell.

Step into Encrypted Methods

The menu command Step into Encrypted Methods sets if the command Step Into (F11) stops in encrypted Methods or not.

  • By default Step into Encrypted Methods is turned off, meaning that the command Step Into does not stop in encrypted Methods.

  • When you turn Step into Encrypted Methods on, then Step Into also stops in encrypted Methods. The Method-Debugger then shows the message The source code is encrypted. instead of the source code as such.
