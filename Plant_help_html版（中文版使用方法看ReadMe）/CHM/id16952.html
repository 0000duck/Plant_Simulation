Calling a Method
2D Reference Help > Information Flow Objects > Method > Calling a Method

You can call a user-defined method:

  • By clicking Run on the Tools ribbon tab in the Method window. Note that you have to click Apply Changes first if you made changes to the source code. If the Method expects parameters to be passed, Plant Simulation opens the Debugger, where you enter the parameters. You mainly use this way of calling a Method while developing your simulation model.

  • By a call from a control from within an object. An entrance control in a Track object may, for example, trigger a Method when an MU enters.

  • By a call from another Method. For this to work, the name of the Method has to be known. If the Method is located in a different name space, you also will have to enter the path.

  • By a call issued by the EventController. The call &method.methcall(time) will start the designated Method after the time in seconds designated by the parameter of data type time has elapsed.

  • By calling a reference. If a variable of data type object contains a reference to a Method, you enter &Variable.

    Example:

    		 
    Method1                  -- same name space
root.Frame2.Method1      -- path and name
&Method1.methcall(8.5)   -- pass to EventController
Variable.execute         -- call the referenced Method, note that the
                         -- object Variable has the data type object
