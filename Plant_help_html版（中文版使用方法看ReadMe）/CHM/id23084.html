_3D.TypeOfAnimations.getAnimation
SimTalk Reference > 3D-related SimTalk Functions > Built-in Methods and Read-Only Attributes for Animations in 3D > _3D.TypeOfAnimations.getAnimation

Syntax:

 
Path._3D.MUAnimations.getAnimation(Index:integer)
Path._3D.SelfAnimations.getAnimation(Index:integer)
Path._3D.CameraAnimations.getAnimation(Index:integer)

The method _3D.TypeOfAnimations.getAnimation returns an animation of the designated type.

Accessing an Animation via the Path Extension

When a saved animation has a name that is a legal SimTalk identifier, you can access this animation via the path extension. This applies, for example, to the Path name > Default of the MU animation. Here you could, for example, type in Assembly._3D.MUAnimations.D. The path extension then suggests

Parameter

The parameter Index of data type integer designates the n-th animation of the animations.

Example:

 
var t: table
self.~._3D.MUAnimations.getAnimation(7).getTable(t)

Example:

 
var t: table
_3D.MUAnimations.Default.getTable(t)

_3D.TypeOfAnimations.getAnimation(Name)

Syntax:

 
Path._3D.MUAnimations.getAnimation(Name:string)
Path._3D.SelfAnimations.getAnimation(Name:string)
Path._3D.CameraAnimations.getAnimation(Name:string)

The method _3D.TypeOfAnimations.getAnimation returns an animation of the designated type.

Parameter

The parameter Name of data type string designates a saved animation, whose name is an illegal SimTalk identifier, "#0#0" in the example below.

Example:

 
var t: table
self.~._3D.MUAnimations.getAnimation("#0#0").getTable(t)

_3D.MUAnimations.getAnimation(IndexX, IndexY)

Syntax:

 
Path._3D.MUAnimations.getAnimation(IndexX:integer, IndexY:integer)

The method _3D.TypeOfAnimations.getAnimation returns an MU animation.

Parameter

  • The parameter IndexX of data type integer designates the x-dimension of the animations of the passed indexed animation point, i.e., of the paths with the notation "#x#y", which, for example, designate the individual processing stations of a ParallelStation or the storage locations of a Store.

  • The parameter IndexY of data type integer designates the y-dimension of the animations of the passed indexed animation point.

    The method MUAnimations.getAnimation(IndexX:integer,IndexY:integer) only applies to MU animations.

    The indexes, which you enter, start at 1, while the indexes of the path names on which they are based start at 0. getAnimation(1, 1) corresponds to getAnimation("#0#0").

Example:

 
.MaterialFlow.ParallelStation.MUAnimations.getAnimation(1, 2).redirectTo("scrap")
-- redirects the animation at the index 1, 2 (#0#1) to the special path "scrap"
