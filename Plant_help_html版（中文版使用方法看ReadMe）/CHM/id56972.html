Toggle Instance Breakpoint
2D Reference Help > Information Flow Objects > Method > The Method Window > Tools Ribbon Tab > Toggle Instance Breakpoint

Click in the line in which you would like to set a instance breakpoint. Then, select the command Toggle Instance Breakpoint or press Shift+ F9 to set an instance breakpoint that only applies to this instance. An instance breakpoint is designated by a red star .

When you insert a class breakpoint and an instance breakpoint in the same line, the Method shows a white star within a red circle .

When Plant Simulationreaches this breakpoint while the Method is being executed, the Method Debugger opens and displays the line inverted in yellow. You can then inspect and modify the contents of the local variables and the parameters.

Instead, you can also click in the line in which you would like to set an instance breakpoint with the right mouse button and select the command Instance Breakpoint on the Context Menu of the Method Debugger.

We mark temporary breakpoints in the source code of a Method in a brighter color than saved breakpoints. This might, for example, be the case when you changed the source code of a Method and set a breakpoint after the original end of the source code. These breakpoints will be lost when you do not apply the source code.

You can also set a class breakpoint and/or an instance breakpoint for an encrypted method. The Method then stops at respectively before method execution in the Debugger.

When you did set a breakpoint, this Method shows the breakpoint with the respective icon within the Method editor or the Method debugger before the message Source code is encrypted.
