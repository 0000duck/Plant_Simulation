Use a Multi-level Experimental Design
2D Reference Help > Tools > ExperimentManager > Controlling the Experiment Runs > Automatically Generating Experiments > Use a Multi-level Experimental Design

Let us assume that to systematically investigate the input values you want to change all values between two boundaries in steps which you define.

The menu command Tools > Multi-level Experimental Design opens the table GeneralDesign. Here you can define a multi-level experimental design. All input values are diversified independently of each other within the boundaries.

The number of interim values results from the increment size which you can enter into the table GeneralDesign for each input value.

The table below results in 3 times 4 times 2 times 2 equaling 48 experiments. The first input value is allocated the values 10, 30, and 50. The range of values of the second input value has four values. As the ExperimentManager creates all combinations of these allocations, this results in a considerable amount of experiments for even a few input values. The ExperimentManager tells you the number of resulting experiments and offers to cancel the experimental design.
