startFailureIn
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Methods of the Material Flow Objects > Methods for the State of the Material Flow Objects > startFailureIn

Syntax:

 
Path.startFailureIn(StartFailureIn:time)

The method startFailureIn fails the object designated by Path.

The method sets the attribute Failed to true. The time interval has to be equal to or greater than zero.

In case the object does not have a failure profile yet, Plant Simulation creates a failure profile with an availability of 100 percent.

When employing the method startFailureIn we recommend to deactivate the internal failure generator by deactivating the check box Active.

When you set the failed state of a material flow object with the attribute Failed, and you would like to import a Service for removing the failure, you have to use the method name of station.failImp.import.

When you requested the service for removing the failure with the method name of station.failImp.import, you have to release the imported service, after the failure has been removed, with the method name of station.failImp.releaseExporters.

Parameter

The parameter StartFailureIn of data type time designates the time after which the failure starts.

Example:

 
MyStation.startFailureIn(10)
