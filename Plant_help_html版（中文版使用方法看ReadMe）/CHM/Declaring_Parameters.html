Declaring Parameters
SimTalk Reference > General SimTalk Functions > Constants and Variables > Parameters > Declaring Parameters

If the Method is to have parameters in a method, you first have to declare these parameters at the beginning of the source code. Start that with an opening parenthesis, then enter the identifier, followed by a colon, a data type and a closing parenthesis. The identifier serves for accessing the parameter later on.

When several parameters in sequence have the same data type, you can use list all identifiers of these parameters, separated by commas, followed by a colon and the data type.

Separate several parameters with different data types with a semicolon. You do not have to enter a semicolon after the last data type, but you have to type the closing parenthesis.

Example:

 
param repeats: integer          -- single parameter of data type integer
param minimum, maximum: real    -- two parameters of data type real
param rpm: real, tool: string   -- two parameters with different data types
param workpiece: string, Islength, shorten: real -- three parameters

In this Method the parameters can be used like Local Variables. The initial value is set by the caller. This way the method can react to different values.

When you enter the keyword byref in front of the parameter, Plant Simulation does not pass the value, but a reference to this variable. Changes of the parameter take effect also for the caller of the method.

For the data types stack, queue, list, and table you just have to enter the type, additional parameters are not required.

Example:

 
param order: list                -- cardfile of any type
param orders, deliveries: queue  -- two queues of any type
param cost: table, VAT: real, complaints: stack -- mixed declaration

If a method does not need any parameters, the declaration data type of the result follows.

Example:

 
-> boolean
param v1,v2: integer, name: string -> boolean
