forget
SimTalk Reference > General SimTalk Functions > Names > Keywords > forget

The keyword forget applies to local variables of data types table, list, queue and stack as well as to type any.

  • When you apply forget to one of the list types, it deletes the local list you created with the method create. Normally you will not use forget because Plant Simulation deletes the list anyway, when the method is finished. You will use it when you want to execute the statement create again.

    Example:

    		 
    var t: table
t.create
t.forget
t.create

  • When you apply forget to a local variable of data type any, it resets this variable, so that you can assign another data type afterwards.

    Example:

    		 
    var a: any
a := "Test"
forget a
a := 123
