_3D.concatenateRotations
SimTalk Reference > 3D-related SimTalk Functions > Built-in Methods for Detailed Access To Graphics > _3D.concatenateRotations

Syntax:

 
basis.~._3D.concatenateRotations(Rotation1:real[4], Rotation2:real[4])

The method _3D.concatenateRotations concatenates two rotations into one and returns the calculated value.

Parameters

  • The parameter Rotation1 is an array of data type real which sets the first rotation of the graphic. The elements of the array denote the rotation angle, and the x-component, the y-component, and the z-component of the rotation axis.

  • The parameter Rotation2 is an array of data type real which sets the second rotation of the graphic. The elements of the array denote the rotation angle, and the x-component, the y-component, and the z-component of the rotation axis.

Example:

 
Station._3D.Rotation := basis.~._3D.concatenateRotations(Station._3D.Rotation, [45, 0, 1, 0])

Example:

 
Station._3D.Rotation := basis.~._3D.concatenateRotations([90, 0, 1, 0], [45, 0, 1, 0])
