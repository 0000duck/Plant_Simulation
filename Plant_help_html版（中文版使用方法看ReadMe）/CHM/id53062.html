Creating Your Own Templates
2D Reference Help > Information Flow Objects > Method > The Method Window > Edit Ribbon Tab > Select Template > Creating Your Own Templates

You can create a method templateof your own in a text editor and save it as a text file (*.txt). The syntax used is quite simple:

  • Create the file in the sub-folder C:\Siemens\Plant Simulation XX\Templates\English\Name_of_Your_Category. The dialog Select Template shows this name in the field below Select the category.

  • Enter two hyphens -- followed by the key combination, which you want to use if you want to set a key combination.

  • Enter two hyphens -- and the name of the template. The dialog Select Template shows this name in the field below Select the template.

  • Enter two hyphens -- and a description of the template. The dialog Select Template shows this description in the rightmost field.

  • Enter /* and then the comment, the template itself shows after you loaded it into the methodeditor. Close this comment with */.

    Enter the variable you want to use, and any comment you would like to add.

    Then enter is and do and any source code you want to provide.

    Enter the item you want the user to replace in the source code within left and right pointing double angle quotation marks, «attribute»=«value» in our example.

    When the user drags the mouse over this item, the Method shows the text you enter at the end of the file as a tooltip: Start this tooltip text with /* followed by the term you entered in the double angle quotation marks. Finish it with §§*/. Note that you can enter the tooltip, but do not have to do it.

  • Enter the key combination that the user can type in to insert the code snippet.

    Proceed as follows to insert the code snippet: Suppose that the key combination is Ctrl+T C. Hold down the key and press the T key on the keyboard. Then release the Ctrl key and type in the letter that is assigned to the code snippet, C in our case.

The text for the processing time depending on an attribute of the part looks like this. 

-- Determines the processing time of a station in a formula
-- according to an attribute of the MU.
-- @ is the MU that triggers the formula
-- ? is the station calling this method
->time

if @.«attribute»=«value» 
   result:=«time»
   -- add your code here
elseif @.«attribute»=«value»
   result:=«time»
   -- add your code here
else
   -- add your code here
end

The Methodshows it like this, after you loaded it.
