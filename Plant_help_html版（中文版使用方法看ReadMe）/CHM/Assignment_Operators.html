Assignment Operators
SimTalk Reference > General SimTalk Functions > Operators and Expressions > Assignment Operators > Assignment Operators

The assignment operator := assigns a new value to a variable. Plant Simulation first evaluates the expression to the right of the operator. If value and variable have the same data type, the value is assigned to the variable.

If the data types are different, the value will have to be converted first. Plant Simulation automatically converts real into integer and integer into real values. Note that during conversion of real values into integer values Plant Simulation deletes the positions after the decimal point. The same is true for assignments of date to dateTime and dateTime to date. The time part will be discarded. 

integer_Var := 0.999 -- value is 0, no digits after decimal point

Note that you will have to explicitly convert all other data types using the respective conversion functions, compare Manually Converting Data Types.

Example:

 
var i: integer, r: real, b: boolean, s: string, d: date, dt: dateTime
i := 5.5                    -- 5
r := 1.0 + 4 * (3 - 0.18)   -- 12.28
b := true and (b or true)   -- true
s := "value"                -- string
s := "new" + " value"                   -- "new value"
d := str_to_dateTime("1.1.98 13:45:44") -- 1.1.1998 no time
dt := d                                 -- 1.1.1998 0:0:0 midnight
                                        -- set time to 0

Plant Simulation first completely evaluates the right hand side of an assignment before assigning the value to the variable on the left side. This allows to directly increase the value of a variable.

Example:

 
a := a + 3 -- integer

Suppose the variable a still has the value 5 assigned to it from a previous example. The result on the right side is formed by the addition of the previous value of a and the integer constant 3. This sum is then assigned to the variable a. The previous value will be overwritten!

You can assign values with the operators & (SimTalk 2.0) ref (SimTalk 1.0) and byref or you can assign void.
