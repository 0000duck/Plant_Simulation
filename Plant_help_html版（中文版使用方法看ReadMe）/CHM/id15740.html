startExtProc
SimTalk Reference > General SimTalk Functions > Predefined Functions > Operating System Functions > startExtProc

Syntax:

 
startExtProc(PathToProgram:string[, Visible:boolean, WaitUntilProcessTerminated:boolean])) → integer

The function startExtProc starts an external process.

If you deactivate the safety setting File > Model Settings > General > Access to the computer is permitted Plant Simulation does not execute the function startExtProc and shows an error message.

Parameters

  • The parameter PathToProgram of data type string designates the external process.

    This external process usually is a program with a graphical user interface that does not have a Console. Plant Simulation itself will continue to be executed while the external process is open.

    Enter a double-backslash \\ as a path separator. Plant Simulation continues being executed while the external process is running. The function either returns the process ID ( PID) or it returns 0 if the function fails.

  • The parameter Visible sets if the started program is to be visible (true) or not (false).

    If you do not specify the parameter, the program window will be visible.

  • The parameter WaitUntilProcessTerminated sets if the function is to wait until the started program was terminated (true) or not (false). If you do not specify the parameter, the function does not wait, i. e., the instructions, which are to follow after startExtProc, will be executed immediately.

    When the command window distracts you from a system command, for example system("dir file.txt"), you can hide the command window with the function startExtProc, for example startExtProc("cmd.exe /C dir file.txt", false, true)

Return Value

The return value has the data type integer.

Example:

 
startExtProc("C:\\Program Files (x86)\\Adobe\\Acrobat Reader DC\\Reader\\AcroRd32.exe")
