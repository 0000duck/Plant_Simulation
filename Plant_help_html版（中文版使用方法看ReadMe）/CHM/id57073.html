ResWorking
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Read-Only Attributes of the Material Flow Objects > Read-Only Attributes for the State of the Material Flow Objects > ResWorking

Syntax:

 
Path.ResWorking → boolean

The read-only attribute ResWorking returns if the object designated by Path is Working (true) or not (false).

Plant Simulation can observe the read-only attribute ResWorking, i.e., you can use it in waituntil-constructs or the object TimeSequence may record it in watch/observe mode.

The definition of working depends on the object doing the processing:

The object

Is working when

Station, ParallelStation, AssemblyStation, DismantleStation, PlaceBuffer, Drain

it processes at least one MU

Track, TwoLaneTrack

at least one MU is moving on it

Conveyor

its current speed is not 0. It can also be working when it is moving and when it does not transport any part

Turnplate, Turntable

it has a speed or when it is rotating, no matter if it moves a part or not

Converter, AngularConverter

has a speed or when the moving time is running

PickAndPlace Robot

it is rotating, no matter if it moves a part or not

Source, Store, Buffer, Transporter, Container, Tank

--

Sorter

--

The processing time is identical with the sorting time, i.e., it is always zero. This means the sorting takes place instantaneously. Even though the processing time is zero, each sorting action counts as a processing procedure.

FluidDrain, FluidSource

material flows between stations

Mixer

when materials are mixed

Portioner

when the is transformed into an MU

DePortioner

when the MU is transformed into a material

Return Value

The return value has the data type boolean.

Example:

 
print MyStation.ResWorking
