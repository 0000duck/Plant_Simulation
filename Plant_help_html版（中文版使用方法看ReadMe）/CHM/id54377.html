Properties of the StackFile
2D Reference Help > Information Flow Objects > QueueFile and StackFile > Properties of the StackFile

The object StackFile is a list with one column that Plant Simulation accesses using the LIFO method (Last In First Out). Plant Simulation inserts new entries at the top of the StackFile and removes the contents of the cell you added last first.

You can access the functions of the list objects on the List Ribbon Tab.

The StackFile always opens in the background behind any open dialog windows. You can also open it in the foreground as a dialog window with the method openDialogBox.

The data type Stack shares the built-in properties of the StackFile.

Note the difference between the object StackFile, which you insert into your model, and the data type stack. You can create user-defined attributes and local and global variables of data type stack that are part of another object and thus are not an object of their own and do not have their own icon.

For this reason these variables and attributes do not recognize the methods of the StackFile, such as location or existsIcon. All other methods, especially for read and write access, apply to both the StackFile and to variables and attributes.
