The menu command Ignore opens a submenu where you select which types of events the Debugger ignores when executing a Method.

Ignore Breakpoints

You can set breakpoints in the Debugger with the commands Toggle Class Breakpoint, and Toggle Instance Breakpoint. Select Ignore Breakpoints to make Plant Simulation ignore these user-defined breakpoints.

Instead, you can also click Ignore Breakpoints on the Debugger Ribbon Tab.

Ignore Errors

The menu command Ignore Errors stops the simulation after an error occurred in the current Method. Plant Simulation opens the Debugger with the Method containing the error and marks the line containing the error in red.

The simulation might continue in case additional events are entered into the events list in the EventController. This way you can continue working, although your model contains methods with faulty code.

When an error occurs, Plant Simulation shows an error message in the status bar. It contains a description of the problem and the Method, the line number of the error and the active call chain

Instead, you can also click Ignore Errors on the Debugger Ribbon Tab.

Ignore Errors in Formulas

The menu command Ignore Errors in Formulas prevents opening the Debugger when a runtime error occurs in a formula.

Instead, you can also click Ignore Errors in Formulas  on the Debugger Ribbon Tab.
