List
SimTalk Reference > General SimTalk Functions > Data Types > List

A variable of data type list shares its built-in properties with the object CardFile.

Note the difference between the object CardFile, which you insert into your model, and the data type list . You can create user-defined attributes and local and global variables of data type list that are part of another object and thus are not an object of their own and do not have their own icon.

For this reason these variables and attributes do not recognize the methods of the CardFile, such as location/ ~ or existsIcon. All other methods, especially for read and write access, apply to both the CardFile and to variables and attributes.

Before first accessing the data type list, you have to apply the method create to the variable or you have to assign a value to the variable.

Example:

 
var l: list[string]
l.create
l.insert(1,"Hello")
