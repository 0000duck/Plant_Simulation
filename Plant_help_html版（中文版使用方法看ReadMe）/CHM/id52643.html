Tab MU Animation

On the tab MU Animation you can define how the parts (MUs) are animated on the material flow objects. Note that the Frame also counts as a material flow object.

The tab looks slightly different for point-oriented objects and for length-oriented objects.

You can:

  • Enter the name of the Animation object and select the MU side to attach.

  • Add, Extend, Edit, and Delete Animation paths.

  • For the length-oriented objects you can add an Animation offset and activate or deactivate Gravity mode.

  • Select settings for testing the animation path.

  • Click Add > Rotation path (lines) to add a rotation path to the point-oriented and to the length-oriented objects.

  • Activate Animation area to create storage places on the object with matrix loading area (ParallelStation, Sorter, Store, Transporter, and Container).

Introducing MU Animation

The MU Animation, i.e., showing the MUs on the material flow objects during the simulation, is entirely controlled by Plant Simulation 2D. This means that you cannot play an animation for a part without this animation being triggered by an event in Plant Simulation 2D.

You can only intervene in animation by defining animation paths and animation points and redirecting an animation onto another one.

The simulation object, i.e., the object on which the part is located in Plant Simulation 2D, determines which animation will be played in 3D by default:

  • When 3D does not find the MU animation path, which is to be used, it does not animate the MU.

  • Conveyor, AngularConverter, Turnplate, Turntable, Track, and FootPath use the path named Default. If no such path exist, an automatic animation runs.

  • The TwoLaneTrack uses the paths named A or B depending on the lane on which the MU moves.

  • The Converter uses the path named Default for lengthwise movements and Cross for crosswise movements.

  • The PlaceBuffer uses this logic one after the other for the MU Animation:

    • When the MU animation area is active, the PlaceBuffer uses it. In this case the index of the part identical with the index of the area in X-direction; the capacity in Y-direction always is 1.

    • Else if an MU animation path named Default with more than an anchor point exists, the index of the part is converted into a relative position; the part are equally distributed on the line.

    • Otherwise the PlaceBuffer uses the first path for the first part, the second path for the second part, etc. If no matching path exists, but if a path named Default exists, which only consists of a single point, the PlaceBuffer uses that path.

  • An object with a matrix loading space uses animation area, if it is activated. If it is not active, it uses the paths getAnimation(x, y) or #<x-1>#<y-1> respectively. This applies, for example, for the ParallelStation.

  • A Transporter with active loading space uses the path named Line.

  • A Transporter with passive loading space uses the path named Track.

  • In all other cases 3D uses the path named Default.

You can redirect the MU Animation to another animation with the method _3D.MUAnimations.AnimationPathName.redirectTo.
