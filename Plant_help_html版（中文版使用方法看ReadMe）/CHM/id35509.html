Watchable Values
SimTalk Reference > General SimTalk Functions > Control Flow Statements > Suspending Methods > Watchable Values

Plant Simulation must be able to observe the conditions used in waituntil- or stopuntil-statements. A condition is observable if all components of the boolean expression are observable. Some examples of conditions that cannot be watched are the return values of user-defined methods and values that change continuously, such as the utilization of stations. When the condition contains a component that Plant Simulation cannot watch, the Debugger will open and display the error message Expression cannot be watched.

When the condition contains a component that Plant Simulation cannot watch, the Debugger will open and display the error message Expression cannot be watched.

The window Show Attributes and Methods shows all methods and attributes that the program can observe.
