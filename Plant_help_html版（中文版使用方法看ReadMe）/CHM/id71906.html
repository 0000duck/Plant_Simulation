Set the Animation Area of an Object in 3D
Step-by-Step Help > Modeling in 3D > Working with Objects > Set the Capacity of a Material Flow Object > Set the Animation Area of an Object in 3D

Whereas you define the capacity of the objects in 2D in two dimensions, you set the animation of the simulation object in 3D in three dimensions on the animation area.

To set the animation area of an object with matrix loading space (ParallelStation, Sorter, Store, Transporter, Container) in 3D, proceed as follows:

  • Click the object with the right mouse button to select it.

  • Select Edit 3D Properties > MU Animation > Animation area.

In our example we demonstrate how to place cylinder-shaped containers in a storage box.

A Source each produces the storage boxes and cylinder-shaped containers. The AssemblyStation places the containers in the storage boxes and then moves them on across a Conveyor and a Station to the Drain.

Proceed as follows to create the simulation model:

  • Insert two Sources. We named them SourceStorageBoxes and SourceCylinders.

  • Insert an AssemblyStation. We selected the settings below:

  • Insert a Conveyor. We used a Conveyor of type Guide-railed belt conveyor.

  • Insert a Station and a Drain.

  • Make sure that all objects are connected with Connectors.

As we need customized MU types for the storage boxes and the cylinder-shaped containers, we duplicate the Part and the Container in the folder MUs in the Class Library.

  • Press the F2 key and rename the duplicated container from Container to StorageBox.

    To configure the storage box:

    • Click the storage box with the right mouse button and open it in 3D.

    • Click with the right mouse button into the background of the 3D window and select Exchange Graphics. We selected the storage box below.

      Click Yes in the message dialog that Plant Simulation shows.

      As you certainly remember the s3D files do not only contain the graphic of the object but also information about its dimensions and animation data.

      If you click No, Plant Simulation does not apply the adjustments of the dimensions of the storage box and scales it to the current size of the MU named Container, as the check box Scale automatically on the tab Transformation is selected by default.

  • Press the F2 key and rename the duplicated part from Part1 to Container. To configure our container:

    • Click den container with the right mouse button and open it in 3D.

    • Click the default graphic of the Part and delete it. We are going to create our own graphic for our container.

      Click the Edit ribbon tab and click Cylinder. Type in the values below. Click Create. Make sure that our container is placed on the origin of the grid.

Run the simulation and watch how the AssemblyStation places 12 containers each in a storage box and then moves the full storage box on.

The animation area of the storage box, which is activated by default, makes sure that the containers are distributed evenly across the bottom of the storage box. In doing so, Plant Simulation automatically uses these settings:

  • Click Show/Hide, to show the animation area in the 3D window to see what it looks like. Plant Simulation shows the animation area proper in red with a red orientation indicator in the center.

  • The orientation > XY plane evenly distributes the containers in the x-y-direction.

  • For the length Plant Simulation only uses 94 % of the length of the graphic of the storage box of 80 cm and for the width only 92 % of the width of 60 cm. Plant Simulation automatically subtracted the wall thickness of the storage box from the dimensions of the animation area. Plant Simulation takes the area settings from the s3D graphic.

  • For the z-position of the center Plant Simulation automatically entered 0.05 m as the floor of the storage box has a height of 5 cm.
