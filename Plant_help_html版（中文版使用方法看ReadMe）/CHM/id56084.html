byref
SimTalk Reference > General SimTalk Functions > Operators and Expressions > Assignment Operators > byref

You can use the keyword byref to pass parameters as a reference. This means that Plant Simulation does not copy the value into the parameter, but that the called method directly accesses the called variable. The keyword byref can return more than one result to the calling method. Note that you can only enter local variables. The data type of the passed variable has to be the exact same data type as the data type of the formal parameter, i.e., Plant Simulation does not convert the data types of referenced parameters.

Example:

 
param byref a,b : real -- declares method1
a := a + 1
b := b + 1
                       -- declares method2
var x, y : real
print x, " ", y -- 0 0
method1(x, y)
print x, " ", y -- 1 1
method1(x, x)
print x, " ", y -- 3 1
