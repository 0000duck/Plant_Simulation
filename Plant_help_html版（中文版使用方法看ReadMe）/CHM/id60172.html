Instantiating Local Lists and Tables
SimTalk Reference > General SimTalk Functions > Data Types > Instantiating Local Lists and Tables > Instantiating Local Lists and Tables

A local variable of data type stack, queue, list and table and lists within lists and tables within tables does not contain the data itself, but a reference to the respective data structure. When the variables are entered into a Method object, they are void, i.e., they do not contain a reference to a data structure yet. You have to create the variable, before you can access the data.

Lists and tables provides the methods create and createNestedList for instantiating them.
