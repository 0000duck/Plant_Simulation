Entering Start Options
Setting-Up and Starting > Starting Plant Simulation > Entering Start Options

Proceed as follows to enter arguments, which Plant Simulation uses when you start it:

  • Create a shortcut for Plant Simulation on the desktop .

  • Open the dialog Properties of the shortcut.

  • Extend the path to Plant Simulation in the text box Target with one or several of these start options.

    To do this

    Enter

    Automatically save the model every t minutes.

    /a <t>, for example /a 10

    Plant Simulation adds - AutoSave to the name of the simulation model

    Set the affinity mask which defines on which CPU cores Plant Simulation is allowed to run. The affinity mask is a number in which each bit represents a core on which the threads of the Plant Simulation process are allowed to run.

    /CPU mask

    Set the current working directory.

    Compare getCurrentDirectory and setCurrentDirectory.

    /cwd dir

    Execute the source code of the text file named file.

    /e file

    Start Plant Simulation and open the file named model file.

    /f model file

    Start Plant Simulation, open the file model file, and start the simulation in the Frame designated by Frame.

    /f model <file /m Frame>

    Hide Plant Simulation after starting it, i.e., minimize it into the system tray.

    Compare hideBBL.

    /hideBBL

    Start Plant Simulation with the license name which you enter. The name of the license is case insensitive, the default is PROFESSIONAL.

    /L PROFESSIONAL, or SIMULATION, APPLICATION, STANDARD, RUNTIME, STUDENT, RESEARCH, EDUCATIONAL, VIEWER

    Write all messages which are output to the Console to the file designated by file name into the folder into which you installed the program.

    Compare clearLogFile and getLogFile.

    /logfile file name

    Use the previous, no longer supported Excel interface for reading and writing files using the xls-format. This allows you to use the Excel interface without Excel having to be installed on your computer.

    /NativeExcel

    Prevent Plant Simulation from creating a backup file ( *.bak) when saving the model.

    /NoBakFile

    Prevent Plant Simulation from opening any message boxes. If the message box shows a question with Yes and No buttons, the question will be answered with No. This is especially useful when you execute batch runs, as it prevents them from stopping, when a message box opens.

    When a runtime error occurs in a Method and when this error is not caught by an ErrorHandler, Plant Simulation normally opens the Method debugger. If you set the start option /NoMessageBox, Plant Simulation opens a message box instead telling you that it will automatically terminate in 5 seconds if you do not click OK in the message box window.

    /NoMessageBox

    Prevent Plant Simulation from writing any of the info and debug messages, such as warnings, to the Console.

    Compare setConsoleFilter.

    /NoWarning

    Register the file extension .spp in the Registry.

    /RegisterFileExtension

    Start Plant Simulationwith the user interface language English, German, Japanese, Chinese, Russian, or Hungarian.

    Compare userInterfaceLanguage.

    /UILanguage:ENU

    /UILanguage:DEU

    /UILanguage:JPN

    /UILanguage:CHS

    /UILanguage:RUS

    /UILanguage:HUN

    Set the code page for correctly displaying the contents of tables in models that were created in foreign languages on a computer with an English or a German operating system. The code pages are: Japanese: 932, Chinese: 936, Hungarian: 1250, Russian: 1251, English or German: 1252.

    Compare setCodePage.

    /Codepage:932

    /Codepage:936

    /Codepage:1250

    /Codepage:1251

    /Codepage:1252

    Start Plant Simulation, load the model designated by file_1, and compare it with the model designated by file_2. As a rule this only makes sense for automated use in source code version control tools.

    /compare <file_2 file_1>

    Start the internal Web server

    Start the internal Web server with the specified port number

    /WebServer

    /WebServer:Port number

    Treat models, which you receive from others as if they were your own models, i.e., models with your trust ID. When you open a model that your received from somebody else, for which the Model setting > Prohibit access to the computer is deactivated, Plant Simulation does not notify you about this.

    Models, which were saved with a version older than 14, do not receive a trust ID and thus are always considered to be models from other people. Normally access to the computer is denied when loading an old model into version 14. If you entered the start option /TrustModels, the old model automatically has access to the computer.

    /TrustModels

    Execute the method of the model loaded with /f specified by <path>

    /E <path>

    Instead of a / ( forward slash) you can also type in a - (minus). You can, for example, type /f MyAssemblyModel.spp or -f MyAssemblyModel.spp, both execute the same command.

    You can also pass any other command-line arguments which you can then query with the function getCommandConveyorArg. Plant Simulation itself will ignore them.

    The start options are case-sensitive.

    You might, for example, enter:

    "C:\Program Files\Siemens\Tecnomatix Plant Simulation 14.1\PlantSimulation14_1.exe" /UILanguage:ENU /f MyAssemblyModel.spp

    into the text box Targetof a shortcut.
