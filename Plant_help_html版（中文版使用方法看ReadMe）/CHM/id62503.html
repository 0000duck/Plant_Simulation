When comparing strings Plant Simulation does distinguish between upper-case and lower-case. To run a case-insensitive comparison, use the operator ~=, described below.

Plant Simulation always compares absolute (object) paths. Plant Simulation resolves relative paths before comparing them.

The operators <, <=, >= ,> compare for greater or smaller. The operators ~=, <~=, >~= compare for about equal, less than or about equal, and for greater than or about equal:

When you compare two strings using , =, >=, or > Plant Simulation does distinguish between upper-case and lower-case. When, on the other hand, you compare two strings using ~=, ~=, or >~=, Plant Simulation does not distinguish between upper-case and lower-case.

You can use the about equal comparison to prevent rounding errors when using real values. Try this:

print (1/3+1000-1000)*3 and watch, which number your computer returns.

You can use the method setEpsilon to set the tolerance threshold for which Plant Simulation considers values to be about equal.

You can use the result of a comparison in nested expressions. You might, for example, concatenate several comparisons with the logical operators AND, OR and NOT.

The source code looks like this:

if (index > 0) and (index = List.Dim) 
   print "index in range"
end
