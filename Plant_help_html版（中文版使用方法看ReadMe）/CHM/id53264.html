Using a Computer-Dependent/Dongle ID License
Setting-Up and Starting > Licensing Plant Simulation > Using a Computer-Dependent/Dongle ID License

For drawing the license, Plant Simulation uses the computer’s ID. For floating licenses this is the ID of the license server computer, for node-locked licenses it is the ID of the local computer. The ID can be either computer-dependent or the ID of a WIBU dongle. Both types of IDs support all license types described above.

You can use one of these IDs:

  • Ethernet ID (MAC address): The network card of your computer provides this computer-dependent ID. The ID only changes when you replace your network card.

    An Ethernet ID might look like this: 001118ba11f0

  • WIBU Dongle ID: A software protection dongle is a piece of hardware which you plug into an USB port of your computer. The dongle ID is located on the dongle itself.

    Using a WIBU dongle enables you to move the dongle and the license files to different computers. This is useful when you replace or reinstall the license server computer, or disconnect it from the network. The portable dongle makes licenses available again quickly and easily.

    A WIBU Dongle ID might look like this: WIBU=1234

  • Composite Host-ID (CID): The Composite Host-ID ( CID) factors in the name of the computer and the network card. Use the first COMPOSITE in the field Query Host-IDs. It might look like this: COMPOSITE=A12345B67899
