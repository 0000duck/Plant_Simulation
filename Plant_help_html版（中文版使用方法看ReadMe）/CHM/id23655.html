TargetPosition
2D Reference Help > Mobile Objects > Transporter > Attributes of the Transporter > TargetPosition

Syntax:

 
MU-path.TargetPosition

The attribute TargetPosition sets the target position of the Transporter designated by Path on a Track. The default value is -1.

This only applies when the Transporter is located all the way on a Track and when it is not a part of a tugger train.

  • If you do not specify a value for the acceleration, the Transporter drives to this position and stops there.

  • If you do enter a value for the acceleration, the Transporter accelerates to the target speed, drives on which a constant speed and decelerates in time so that the Transporter stops a the target position. When the distance to the target position is too small to reach the target speed and to decelerate, Plant Simulation uses the highest possible speed.

The Transporter automatically turns around when a change of direction is required to reach the target position. When you stopped the Transporter before setting the target position, it also starts automatically.

The attribute is observable. When reaching the target position, Plant Simulation calls the Destination control.

If you want to stop a possible driving towards the target position, you can set the attribute Stopped to true. Then Plant Simulation stops the Transporter without it reaching the target position. Plant Simulation does not call the Destination control and sets the value of the attribute TargetPosition to the default value -1.

Setting this attribute influences other attributes, such as Stopped and Backwards. It then also deactivates inheritance.

Data Type of the Value You Can Assign

You can assign a value of data type real.

Example:

 
.MUs.Transporter:1.TargetPosition := 8
