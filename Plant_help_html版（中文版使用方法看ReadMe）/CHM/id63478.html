Backwards
2D Reference Help > Mobile Objects > Transporter > Attributes of the Transporter > Backwards

Syntax:

 
MU-path.Backwards

The attribute Backwards makes the Transporter drive in reverse on the objects Track or TwoLaneTrack. When the Transporter moves from a Track/TwoLaneTrack to a Track/TwoLaneTrack, it may enter the Trackat the Track’s exit and leave it at its entrance. This only applies when the Transporter moves from a Track/TwoLaneTrack to a Track/TwoLaneTrack, but not from a point-oriented object onto a Track/TwoLaneTrack.

When you deactivated the check box Acceleration or set the attribute AccelerationEnabled to false, the attribute Backwards (Backwards) sets if the Transporter designated by MU-path moves in reverse (true) or forward (false) on the Track. If, on the other hand, the feature Acceleration is active, the attribute Backwards marks the sign, positive or negative, of the final speed of the Transporter. Suppose a Transporter drives forward on a Track with a certain speed with the check box Backwards deactivated. Then you select the check box Backwards. This will result in the Transporter slowing down until its speed is 0. Then it accelerates in reverse direction until it reaches its final speed.

Data Type of the Value You Can Assign

You can assign a value of data type boolean.

Example:

 
if @.empty 
   @.backwards := true
else
   waituntil SP2.empty prio 1
   @.Cont.move(SP2)
   @.backwards := true
end

Example:

 
.MUs.MyTransporter.Backwards :=true
