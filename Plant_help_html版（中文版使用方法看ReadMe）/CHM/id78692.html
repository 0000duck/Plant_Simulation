Methods of the Transporter
2D Reference Help > Mobile Objects > Transporter > Methods of the Transporter > Methods of the Transporter

The Transporter provides:

  • The methods endFailure, endFailureIn, endPause, endPauseIn, eraseFailureEvents, erasePauseEvents, findFreePlace, getLane, getLaneNo, getRouteLength, getRouteToDestination, hitchFront, hitchRear, onLaneA, onLaneB, setRoute, startFailure, startFailureIn, startPause, startPauseIn, unhitchFront, and unhitchRear.

  • The Methods of All Mobile Objects.

  • The Methods of All Objects.

  • The Methods of the Material Flow Objects.

To view all of the methods and attributes of the Transporter, open the window Show Attributes and Methods. The figure below illustrates the information using the example of the object Source.

The dialog Show Attributes and Methods, which you can open by pressing the F8 key, only shows the attributes and methods pertaining to the selected type of loading space. If you select Track for example, the dialog only shows the attributes and methods pertaining to the loading space Track, not those pertaining to the loading space Store or Line.

  • Select Show Attributes and Methods on the context menu of the Class Library to show the attributes and methods of the selected Class.

  • Press the F8 key or click Show Attributes and Methods on the Home ribbon tab of the Frame into which you inserted an instance to show the attributes and methodsof the selected Instance.

An example of the Syntax line of the individual methods might look like this:

Syntax:

 
MU-path.create(MU_location:length[,Position:length, CopyStatistics:boolean])

  • MU-path designates a valid path to the MU (for example: MyTrack.Cont, ParallelStation.MU(1), .MUs.Part:203). If you use the Method as an entrance or exit control, you can access the MU that triggered the control using the anonymous identifier @(see Anonymous Identifiers).

  • MU-location designates a location- or length-oriented material flow object, a single processing station of an object with several stations, or a length-oriented object including a position. You can enter the position as the additional parameter of data type length. If you do not enter a position, Plant Simulation uses 0.

  • The signature of the method, consisting of the identifier and the data type of the parameter, is listed in parentheses. The expression (Parameter:real), for example, designates a parameter of data type string. Instead of a constant value, you can also use a variable of the required type or a method that returns the required data type.

    Optional parameters are listed within square brackets. The expression [,Parameter:boolean], for example, means that you can, but do not have to enter the boolean parameter.

    Make sure to enter the parentheses for expressions within parentheses (…). Not entering them may lead to unexpected results and open the Debugger.
