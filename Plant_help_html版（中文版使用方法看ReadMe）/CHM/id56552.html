In this example we track the course the Part with the ID 9 takes through a simulation model and write the stations to a trace file.

With the settings shown in the dialog above, Plant Simulation inserts a breakpoint for each Out event of the Part with the ID 3 and writes this event to the file c:\Exercises\trace3.txt. Note that this only works when you selected the check box Trace active in the dialog Event Debugger. When you clear the check box Breakpoints active, Plant Simulation writes the trace file without stopping your simulation using a breakpoint.

In the examples above we only used Out events. You can, of course, also use any of the available types of events, compare the List of scheduled events.
