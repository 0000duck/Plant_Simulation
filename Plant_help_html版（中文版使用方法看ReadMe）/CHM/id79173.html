_3D.optimizeByVisibilityFilter
SimTalk Reference > 3D-related SimTalk Functions > Built-in Attributes and Methods for Accessing Graphics > _3D.optimizeByVisibilityFilter

Syntax:

 
_3D.optimizeByVisibilityFilter(GraphicPath:any[], CullGranularity:integer, ReductionLevel:integer[, GraphicGroupName:string])

The method _3D.optimizeByVisibilityFilter optimizes the graphic designated by Path i.e., it removes graphic components which are not visible from outside.

Parameter

  • The parameter GraphicPath is an array of data type any which sets the path to the graphic. The parameter is described under The Parameters GraphicPath and GraphicGroupName.

  • The parameter CullGranularity of data type integer sets the granularity for deleting graphics. 0 is very fine granular, 3 is very coarse granular.

  • The parameter ReductionLevel of data type integer sets how close the procedure looks at the graphic. 0 stands for a rather coarse look which causes a lot of graphics to be deleted very quickly, 3 stands for a very close look which takes a lot longer.

  • The parameter postProcessing of data type boolean sets if post-processing of the simplified graphic takes place (true) or not (false). Among others post-processing removes isolated triangles.

  • The optional parameter GraphicGroupName of data type string designates the name of the graphic group you are working with. The parameter is described under The Parameters GraphicPath and GraphicGroupName.

Example:

 
.Models.Frame._3D.optimizeByVisibilityFilter([2], 0, 1)

Example:

 
.Models.Frame._3D.optimizeByVisibilityFilter([2], 0, 1, "deco")
