getRouteToDestination
2D Reference Help > Mobile Objects > Transporter > Methods of the Transporter > getRouteToDestination

Syntax:

 
MU-path.getRouteToDestination(DestinationObjects:table) → boolean

The method getRouteToDestination returns if the Transporter did find a route to its destination (true) and writes the objects along the route into a table.

The method getRouteToDestination always returns the last calculated route. Even when the Transporter drove on in the meantime, the route still starts with the original starting point. Plant Simulation only computes the route anew, when the Transporter leaves the pre-calculated route. When the destination of the Transporter is a sensor, and the Transporter has passed this sensor, it still drives on the previous route. Only when the Transporter leaves the Track on which the sensor is located, Plant Simulation computes the route anew.

When Plant Simulation computes the route to the destination, the route starts on the side of the Transporter which is located in the front in the direction of motion. When you entered the destination into a rear-triggered sensor and the front of the Transporter has already passed the sensor, then Plant Simulation disregards that the rear of the Transporter will still pass the sensor.

Parameter

The parameter DestinationObjects of data type table designates the name of the table.

Return Value

The return value has the data type boolean. It returns false if it did not find a route.

Example:

 
MUs.Transporter:1.getRouteToDestination(objectsOnRoute1)

Example:

 
var a: any
if .MUs.Transporter:1.getRouteToDestination(MyTableFile) 
a := MyTableFile.asObject(1,1)
if isObject(a) 
print "first object: ", a.name
else
print "first object is a two-lane track: ", a.~.name
end
print "the length is: ", a.length
