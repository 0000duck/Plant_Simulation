Restore the Original Model Configuration
2D Reference Help > Tools > ExperimentManager > Defining the Experiments > Restore the Original Model Configuration

If you select under Tools > Advanced Settings > Settings, the ExperimentManager will restore these input values after each experiment run. By default this setting is deactivated.

The ExperimentManager recognizes if the current values of the input values differ from the saved values when you start an experiment run and shows a warning message. You can then decide if you want to start the experiment run without saving the values or you can save the values by selecting the menu command Tools > Store Input Values. The method storeParam of the ExperimentManager triggers this action.

The menu command Tools > Restore Input Values writes the values saved in the ExperimentManager back to the model. This way you can return to the original model configuration after an unwanted interruption of the experiment run, caused, for example, by a debugger call. The method restoreParam of the ExperimentManager triggers this action.

You do not need this functionality for a distributed simulation as the controlling ExperimentManager does not modify the model within a distributed simulation.

When you selected the check box Restore original input values for a non-distributed simulation, the input values are saved before the first simulation run of the experiment run in the ExperimentManager and are written to the model again after the last simulation run. To set the saved original input values at the end of the experiment run, the ExperimentManager executes a Reset . This Reset is, for example, required to set buffer capacities.
