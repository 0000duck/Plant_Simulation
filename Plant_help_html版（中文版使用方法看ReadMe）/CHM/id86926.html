The Teamcenter interface is the interface between Plant Simulation and Teamcenter for accessing and exchanging data.

To add the Teamcenter interface to the Class Library, click Manage Class Library > Basic Objects > InformationFlow on the Home ribbon tab.

The Teamcenter interface supports Teamcenter versions 11 and higher and Active Workspace version 3.2 and higher.

To use the Plant Simulation-Teamcenter-Interface the .NET Framework 4.0 needs to be installed on your computer. In addition, the matching Teamcenter client or the Teamcenter Client Communication System (TCCS) needs to be installed on your computer.

The Teamcenter Server requires Plant Simulation files that you have to install.

You can use the commands on the File menu to save your models to Teamcenter, to check these out and edit them, and to check-in the updated version back to Teamcenter.

Saving simulation models in Teamcenter is especially helpful for large companies. It allows the common storage, management, and versioning of the models in projects with specific access permissions. It also allows to use the same system for planning and administering process data, resources, and products.

In addition Plant Simulation and other simulation tools can access the data stored in Teamcenter, enrich them with simulation results, and show them in the simulation report. Other users, managers, etc. can also view the data from Teamcenter.

It also allows to pass models and submodels on to your colleagues and to develop them in groups and to integrate and manage models from suppliers.

If a newer version of the checked in model is located in the Teamcenter database than on you computer, the Teamcenter Interface downloads this newer version so that you can continue working with it.

The figure below gives a general overview about data exchange between Teamcenter and Plant Simulation. It shows Teamcenter processes green and Plant Simulation processes orange.

Teamcenter provides product data, process data, and working area data. Your Teamcenter colleague assembles this data via the Collaboration Context in Teamcenter. He then exports a snapshot via the application interface, which you can then select in Plant Simulation under Application interface. The exported data is transformed using the defined Style sheet to a simplified XML format, which you can then import into a Plant Simulation table. You can then use this data, together with the JT files that were exported from Teamcenter, as input data for your simulation models in Plant Simulation.

The Plant Simulation installation package, which you downloaded, contains the folder Teamcenter Interface. Your Teamcenter administrator has to install the files in the folder TC11 TEM Package with the Teamcenter Environmental Manager. This folder contains the data types for the Plant Simulation Integration, such as the application interface, etc.
