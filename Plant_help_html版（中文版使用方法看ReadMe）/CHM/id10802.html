Occupied
2D Reference Help > Material Flow Objects > Shared Properties of the Material Flow Objects > Read-Only Attributes of the Material Flow Objects > Read-Only Attributes for the State of the Material Flow Objects > Occupied

Syntax:

 
Path.Occupied → boolean
Path[X:integer, Y:integer].Occupied → boolean

The read-only attribute Occupied returns if the object designated by Path contains at least one MU in whole or in part (true) or not (false).

Return Value

The return value has the data type boolean.

Example:

 
if not MyStation.Occupied then
   @.move(Station)
end

For objects with more than one station, such as the ParallelStation, you can use the read-only attribute Occupied to find out if a specific station contains MUs (true) or not (false).

Parameters

  • The parameter X of data type integer designates the x-coordinate of the processing station.

  • The parameter Y of data type integer designates the y-coordinate.

Example:

 
-- search for an unOccupied station on the ParallelStation and place the MU
for var x := 1 to ParallelStation.xDim 
    for var y := 1 to ParallelStation.yDim 
       if not ParallelStation[x,y].Occupied 
          @.move(ParallelStation[x,y])
          return  -- exits the method
       end
