interSection
2D Reference Help > Information Flow Objects > Shared Properties of Lists and Tables > Methods of Lists and Tables > Methods for Accessing Lists and Tables > interSection

Syntax:

 
Path1.interSection(List:path) → list
Path1.interSection(Range:range, List:path) → list
Path1.interSection(Range:range, Range:range) → list

The method interSection returns those values that the ranges or the lists/tables designated by the parameters share. Both ranges or lists/tables have to have the same data type. The return value is a list with one column containing the shared items. Note that values may be listed more than once.

The method applies to the objects StackFile, QueueFile, CardFile, TableFile, and TimeSequence.

Parameters

  • The parameter Range of data type range designates a range of the table.

  • The parameter Range of data type path designates a list/table.

Return Value

The return value has the data type list.

Example:

 
-- the second column of the tablefile contains numbers 2,4, ... 40 
-- the third column contains the numbers 3,6, ... 60. The cardfile
-- contains the square numbers from -10 to 10.
destination.delete
destination.insertList(2,1,MyTableFile.interSection(cardFile))
-- the second column of the tablefile destination 
-- contains the numbers 4,9,16,36,36
destination.insertList(3,1,MyTableFile.
interSection({2,1}..{3,2},cardFile))
-- the third column of the tablefile destination contains 4
dest.insertList(4,1,MyTableFile.interSection (2,1}..{2,*},{3,1}..{3,*}))
-- the fourth column of the tablefile 
-- destination contains the numbers 6,12,18,24,30,36
