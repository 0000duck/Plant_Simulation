Value
2D Reference Help > Information Flow Objects > Variable > Dialog Window of the Variable > Tab Value > Value

Depending on the Data type you selected, you can:

  • Select the value from a drop-down list.

  • Enter the value of the Variable.

    For the data types length, weight, and speed the units you select in the Plant Simulation main window under File > Model Settings/Preferences > Units > Mass /Speed/Length determine how Plant Simulation interprets the value. A value of 100 with the length set to km is interpreted as 100 km.

  • Click Open to open the dialog window of the objects that correspond to the data types table, stack or queue.

  • Click and select the object of data type object in the dialog Select Object.

  • You can also enter the path to and the name of the object into the text box Value.

    To open the dialog of the object, whose name you entered into the text box, press the F2 key.

    The data type object designates an absolute or a relative path to an object.

    • The absolute path always begins with a period, starting in the Class Library, such as .building and identifies the object in a unique manner. Note that Plant Simulation does not modify the absolute path when you derive or insert a Frame employing a Variable object, i.e., the absolute path will still reference the source object, not to the one you derived or inserted!

    • The relative path, which starts with location or ~, considers the current location of the Variable object each time the object Variable is accessed. Thus, when deriving or inserting an object consisting of several hierarchically structured Frames , you do not have to modify the existing object values in the Variable objects.

  • If you would like to use the data type time with a random number distribution, select the data type randtime, and enter the parameters of the distribution function, as the data type Time only accepts constant values. Note you have to initialize the randtime value before accessing it, for example by employing the method rollDice, which determines the next random number the Variable uses.
