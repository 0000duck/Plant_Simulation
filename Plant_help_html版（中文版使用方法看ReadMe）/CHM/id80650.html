Queue
SimTalk Reference > General SimTalk Functions > Data Types > Queue

A variable of data type queue shares the built-in properties of the object QueueFile, compare Properties of the QueueFile.

Note the difference between the object QueueFile, which you insert into your model, and the data type queue. You can create user-defined attributes and local and global variables of data type queue that are part of another object and thus are not an object of their own and do not have their own icon.

For this reason these variables and attributes do not recognize the methods of the QueueFile, such as location/ ~ or existsIcon. All other methods, especially for read and write access, apply to both the QueueFile and to variables and attributes.

Before first accessing the data type queue, you have to apply the method create to the variable or you have to assign a value to the variable.

Example:

 
var q: queue[string]
q.create
q.push("Hello")
