open
Add-Ins Reference Help > Database Interfaces > SQLite > Methods of the SQLite Interface > open

Syntax:

 
Path.open[ReadOnly:boolean := false]

The method open opens the database. You have to open the database before any SQL command can be executed.

The method does not have a return value.

When you open a file-based SQLite database, Plant Simulation applies the following default settings, which deactivate transaction support:

PRAGMA locking_mode=EXCLUSIVE

PRAGMA journal_mode=NORMAL

PRAGMA synchronous=OFF

These settings considerably accelerate access to the database.

To get the SQLite default settings, which support transactions, you can use the following instructions: 

SQLite.exec("PRAGMA locking_mode=NORMAL")
SQLite.exec("PRAGMA journal_mode=DELETE")
SQLite.exec("PRAGMA synchronous=FULL")

To speed up database performance, we recommend to use the following settings instead:

SQLite.exec("PRAGMA journal_mode = MEMORY")
SQLite.exec("PRAGMA locking_mode = EXCLUSIVE")
SQLite.exec("PRAGMA synchronous = OFF")

For more information, consult http://www.sqlite.org.

Parameter

The optional parameter ReadOnly of data type boolean sets if the database can be opened in read-only mode (true) or can also be written to (false). The default value is false. This way Plant Simulation can read the database even when the function Access to the computer is prohibited is active.

Example:

 
MySQLite.open
