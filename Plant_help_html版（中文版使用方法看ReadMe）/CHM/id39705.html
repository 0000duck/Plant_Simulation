Toggle Class Breakpoint
2D Reference Help > Information Flow Objects > Method > The Method Window > Tools Ribbon Tab > Toggle Class Breakpoint

Click in the line in which you would like to set a class breakpoint. Then, click Toggle Class Breakpoint or press the F9 key to set a class breakpoint in front of the statement of the line. A class breakpoint is designated by a filled red circle .

When you insert a class breakpoint and an instance breakpoint in the same line, the Method shows a white star within a red circle .

When the Method reaches a breakpoint while being executed, the Method Debugger opens and highlights the line in yellow. You can then inspect and modify the contents of the local variables and the parameters.

Instead, you can also click in the line in which you would like to set a class breakpoint with the right mouse button and select the command Class Breakpoint on the Context Menu of the Method Debugger.

Or you can click on the Toolbar of the Method Debugger

We mark temporary breakpoints in the source code of a Method in a brighter color than saved breakpoints. This might, for example, be the case when you changed the source code of a Method and set a breakpoint after the original end of the source code. These breakpoints will be lost when you do not apply the source code.

You can also set a class breakpoint and/or an instance breakpoint for an encrypted method. The Method then stops at respectively before method execution in the Debugger.

When you did set a breakpoint, this Method shows the breakpoint with the respective icon within the Method editor or the Method debugger before the message Source code is encrypted.
